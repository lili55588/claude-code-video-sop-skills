## 一、硬规则（违反即退回，无例外）

| 规则 | 定值 |
|---|---|
| 单分镜上限 | **10s**（整数秒、向上取整）。逃生口：仅在用户**明示愿意冒险**时可放至 15s |
| 景别基础时长 | 特写 1~3s / 近景 2~4s / 中景 2~4s / 全景 2~6s |
| 语速 | 每秒 5 字（含标点），说话时长 = 字数 ÷ 5 |
| 分镜计算时长 | max(说话时长, 动作时长) |
| Clip 切分 | 4~10s，装满优先，8~10s 即封箱 |
| 全片策略 | ≤10s：1 任务整段直出；>10s：拆分镜**一次性并行**提交（禁串行/分批） |
| Prompt 结构 | 8-A 黄金结构（即梦官方规范） |
| 参考素材引用 | 参考主体用 `<subject>主体名</subject>`；普通参考图用 `@图片N`，必须附素材顺序表 |
| 环境人数状态 | Phase7 空场景图只锁空间/光线/材质，不代表最终无人；Phase5 每镜/Clip 必须写 `环境人数/背景群众状态`，Phase8 每个分镜正文必须显式写无人物/只有命名角色/背景学生路人乘客人群等状态 |
| 人口声明措辞（两版词库互通，固定写法防漂移） | **明确无人**优先写 `无人物` / `空无一人` / `没有人` / `只剩{命名角色}`，少用单独的"无旁人"（改"无其他人物"）；**有背景人**固定写 `背景有{身份+密度+状态}，背景群像、景深虚化、不抢主体`（如"背景有满座低头自习的学生"）；放学后/打烊等半空场景写 `空座为主、仅余少数{身份}在背景{动作}`，不写模糊的"大半座位空着" |
| 多人引用 | 逐个写全名（`主人公@图片1` + `访客@图片2`），禁止合并写 `@图片1的两个人` |
| 代称检查 | 包括通用名词（女孩/男孩/女人/男人/老人/小孩）；前置排除非代称词（吉他/维他命） |

## 二、翻译纪律（分层）

- **Phase 1~3（创意/大纲/剧本）**：创作层，**允许**改写、润色、再创作
- **Phase 5（分镜）/ Phase 8（生成 Prompt）**：下游执行层，**严格无损翻译**，不增不减不演绎

## 三、交付闸（中立裁判）

任何一方产出 Phase 8 Prompt 后，**必须**跑机检校验器并全 PASS 才算交付：

```
python "C:\Users\Administrator\.claude\skills\video-sop\scripts\validate_phase8_prompt.py" <Prompt集文件路径>
```

校验 FAIL 的产物退回产出方修改，另一方不接 FAIL 件。

**两版严格度差异与交付准绳（2026-06-16 交叉评审定，第四轮校正）**：场景人口检查两版**机制不同、不是同一把尺子的松紧档**——
- **Claude 版 `--scene-population`（默认 WARN，不阻断）**：触发条件是「分镜命中公共场景**关键词**（教室/餐厅/站台/商场/超市…）**且**缺人口词」才提示。**关键缺陷（边界样例实测确认）**：词库**未收录**的公共场景（如**办公室、菜市场**）→ **完全无提示直接放行**，不是 WARN；私人/户外场景（书房/林荫道）也不查。即它只能抓"已知公共词"，抓不全。
- **Codex 版 `--require-population-state`（FAIL，阻断）**：**每个分镜无差别**都必须出现人口状态词，否则 FAIL（含书房"只有陈默成年一人"、空镜"无人物"）。覆盖最全、最严。
- **交付准绳＝Codex strict**：Phase8 产物交付前**必须**跑通 `--require-image-each-shot --require-population-state --fail-pronouns` 全 PASS 才算交付。Claude 主笔时可先用本版 WARN 快跑迭代，但**交付前一律以 Codex strict 复检为准**。
- ⚠️ **未来若给 Claude 版加 `--fail-population`，必须等价 Codex strict（每分镜都要人口状态），不能只把"现有公共场景 WARN 升级成 FAIL"**——否则是**假对齐**：Claude strict 只卡餐厅/教室，Codex strict 还卡书房/林荫道/单车棚，同一产物两版 strict 仍打架。
> 已知技术债：两版人口词库（Claude `SCENE_CROWD_RE`+`SCENE_PUBLIC_RE` / Codex `POPULATION_RE`）各自手维护、会漂移（Claude 的 `SCENE_PUBLIC_RE` 当前漏 办公室/菜市场/候诊室/网吧 等公共场所）；新增场景词或人口措辞时两边都要同步。彻底根治需抽共享词表，但违反"各动各的 skill"，暂记不做。
> PASS 计数两版粒度不同（Claude 165 / Codex 163，差在"文件非空/语言规则"等项是否单列），属输出粒度差异、非检查覆盖差异，不影响 RESULT 判定。

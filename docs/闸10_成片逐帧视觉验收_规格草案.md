## 0. 一句话定位

现有闸 1–9 全是**出片前文本/Prompt 闸**（含正则兜底 `validate_phase8_prompt.py`），**没有任何一道在视频真渲染出来后看像素**。闸 10 补洞：每个 Clip 视频生成后、进 Phase9 剪辑交接/用户确认前，**逐帧取证、对照分镜 Prompt 原话，核验人物朝向 / 站位 / 道具 / 跨镜一致 / 身份 / 时长比例**。这是工作流**第一道模型相关（必须用视觉看图）的闸**。

## 1. 总体架构（三层 + 一道复审）

```
Part A 确定性证据工具 frame_audit.py（不判分，只摆证据）
   A1 全帧底座     —— 原生帧率全帧落盘，供差分与 drill-down
   A2 差分路由     —— 逐帧像素差/场景突变打分 → 真实切点 + 突变帧清单
   A3 分镜锚定取证 —— 解析 Prompt 集 → 入/中/出 + 切点对 + 突变帧，配 Prompt 原话
        ▼
Part B 视觉判定（主判）—— 读取证帧，逐分镜逐维度填四态 + 证据
        ▼
Part C 对抗性独立复审 —— 独立盲审同一批取证帧，分歧逐格升级再裁
        ▼
{项目名}_成片逐帧自检报告.md（全 PASS / 列 FAIL_BLOCKED）
```

- **Part A 确定性**：同输入同输出，可被 Codex 任意语言重写兼容版。**永不下"对不对得上"的视觉结论**。
- **Part B/C 模型相关**：判定靠视觉；反糊弄（§7）+ 对抗复审（§8）专治"觉得还可以就说没问题"。

## 2. 触发与前置（自动）

- **自动触发**：任一 Clip 成片视频到手后，**回报该 Clip"已接受"/ 进 Phase9 / 出剪辑交接单之前，必须先过闸 10**。不靠用户开口。

## 4. 抽帧与取证策略（全帧底座 + 差分路由 + 分镜锚定，三层合一）

### A1 全帧底座（标准流程，非判据）
- 原生帧率全帧落盘 `frames\frame_%06d.jpg`，作**可追溯证据底座 + drill-down 入口**，也供 A2 计算差分。
- **完整性语义（v0.3 并入 Codex round-2）**：全帧落盘 + 差分 + 分镜映射构成**前置完整性闸**。未满足 → 状态 `AUDIT_INCOMPLETE / 审片不完整`：**硬阻断**（禁 PASS、禁进 Phase9、禁说 Clip 可用），但**不等于 `FAIL_BLOCKED`**——`FAIL_BLOCKED` 只判"视频内容有错"（朝向错/道具漂移/身份跑脸/比例错/截断），`AUDIT_INCOMPLETE` 是"证据没凑齐、还没资格下判"。两者返修路径不同：`AUDIT_INCOMPLETE` → 修工具/重抽帧后再审；`FAIL_BLOCKED` → 改视频/回退重生成。理由：真正有判别力的是**全分辨率取证帧 + 差分疑点**，不是"盘上有没有 219 张"，但"没凑齐证据"必须强阻断、不能弱化成 WARN。

### A2 确定性差分路由（本版核心增量，两版都缺）
在全帧上跑**确定性**信号（不判分）：
- 逐帧像素差（相邻帧 MAD / 1−SSIM）或 ffmpeg 场景突变打分，得到**运动/突变曲线**。
- 自动标三类**疑点帧**喂给视觉层：
  1. **真实切点 ≠ 声明分镜边界**：突变尖峰落在 `分镜N: a-b s` 之外 → 切点漂移/意外切；用它**确定性定位真实切点**，替代靠肉眼猜时序。
  2. **分镜内局部尖峰**：声明无动作处突变 → 道具弹出/morph/突然移位。
  3. **异常低动**：Prompt 说有动作处却几乎不动 → 冻结/动作缺失。
- 这层让"全帧"真正值回票价，并堵住 Codex 担心的"采样之间漏掉"——比"盯 219 张缩略图"和"lean 靠运气"都强。

### A3 分镜锚定取证帧（替换 Codex 的均匀 fps 采样）
解析 Prompt 集，按**声明分镜**而非均匀时间取**全分辨率**取证帧：
- 每分镜 **入(a+ε)/中/出(b-ε)**（ε≈0.08s，避切点黑场）。
- **切点帧对**：分镜 i 出帧 ⊕ 分镜 i+1 入帧；跨 Clip 则前片末 3-5 帧 ⊕ 后片首 3-5 帧。本机像素横拼，禁 AI 重绘。
- **A2 标出的疑点帧**一并拉成全分辨率取证帧。
- 顺带产低 fps 缩略图拼版仅作**导航**（不是审片单元）。
- 把每个分镜该满足的 Prompt 原话**自动配到**其取证帧组，判定列留空（§11 模板）。

## 5. 确定性硬核对（Part A，ffprobe，不用眼）

- **时长** vs Prompt 标注：差 >0.3s 至少 `ACCEPTED_WITH_RISK`；整段截断/动作缺失 → `FAIL_BLOCKED`。
- **比例/分辨率/帧率** vs Phase6 全局 + 申请生成参数：比例错 → `FAIL_BLOCKED`（除非显式标"仅测试可接受"）。
- 首尾异常黑帧/纯色帧 → WARN。

## 6. 检查矩阵（逐维度，含判定/证据格式/常见 FAIL）

> 每项写证据：`帧文件@时间戳：实际观察 vs Prompt原话 → 判定`。

**6.0 维度总览**（核心 8 维 + 吸收 Codex 5 维）

| 维度 | 来源 | 一句话判定 |
|---|---|---|
| 分镜对位 | 两版 | 真实切点（A2 定位）与声明分镜时段偏差 >0.5s → FAIL |
| 人物朝向（重点） | 两版 | 正/侧/背 + 画左/画右 对齐原话；写了转向要看到过程不瞬切 |
| 站位漂移 | 两版 | 左/中/右 + 前/后景 + 相对门窗桌位置对齐；无动作位移=漂移 |
| 道具漂移 | 两版 | 承载面/容器/归属手/开合/数量对齐；无搬动变位=漂移 |
| 跨镜一致 | 两版 | 切点帧对：服装/伤痕/道具/站位/朝向/光向无解释跳变 → FAIL |
| 身份不跑脸 | 两版 | 脸/发型/身材对齐 @图片N；多帧间不跑脸 |
| 锚点是否被执行 | 两版 | 成片帧 vs Phase8.5 宫格构图/站位/朝向/道具 |
| 时长·比例 | 两版 | §5 确定性硬核对 |
| **动作节拍顺序** | 吸收 Codex | 可见动作顺序匹配分镜，无跳拍/重复/臆造 |
| **场景人口** | 吸收 Codex | 该有背景群像的场景每帧都在，空镜确实无人 |
| **口型对台词** | 吸收 Codex | 有台词镜头嘴动与台词同步、非默片或乱动 |
| **收尾 pose / 多人转身顺序** | 吸收 Codex | 结尾姿态与多人转身次序符合分镜 |
| **时间·光向·地标 / 镜头语言矛盾** | 吸收 Codex | 时段/光向/主要地标在位；景别·机位·运镜不与 Prompt 矛盾 |

**6.x 重点维度证据示例**
- 朝向：`Clip2_s3_出帧@5.92s：人物正脸朝镜头 ✗ Prompt『侧身面向画左的窗』→ FAIL_BLOCKED`
- 跨镜：`Clip2出帧右肩渗血 / Clip3首帧渗血消失且无处理动作 → 状态跨镜断裂 FAIL_BLOCKED`
- 道具：`Prompt 铅笔盒在旧木桌；出帧无搬动却在纸箱里 → 漂移 FAIL_BLOCKED`
- 分镜对位：`A2 测得真实切点在 4.1s，声明 分镜2:3-6s → 晚切1.1s，时序漂移 FAIL`

## 7. 反糊弄判定协议

1. 默认存疑：每项先假设有漂移，拿正证据才判 PASS。
2. PASS 也要给正证据：禁"看起来一致/应该没问题"，必须帧+时间戳+实际观察+原话。
3. 怀疑项先穷举：每 Clip 先列所有"哪怕一点点不对"，再逐个裁。
4. 二元勾选表，不写散文。
5. **前置完整性闸**：全帧/差分/分镜映射未齐 → `AUDIT_INCOMPLETE`，硬阻断（禁 PASS、禁进 Phase9），先补审片再判，不得弱化为 WARN。
6. 质量判定四态（前置闸过后才给）：`PASS / ACCEPTED_WITH_RISK(写明风险) / FAIL_BLOCKED / REGENERATED_PASS`。
7. 阻断：任一维度 FAIL_BLOCKED → 该 Clip 不进剪辑，按 Phase9 返修/判责树。
8. 看不清只能降级（ACCEPTED_WITH_RISK/待人工复看），**不准默认 PASS**。

## 8. 对抗性独立复审（Part C）

1. **主判**：主对话读取证帧 + 工作表，按 §6/§7 填四态 + 证据。
2. **独立盲审**：派独立子代理，**只给取证帧 + 各分镜 Prompt 原话 + 对抗 brief**，不给看主判结论（防锚定）。brief 要点：默认有漂移、任务是找哪里对不上、死磕朝向/切点对/道具归属手、禁"基本一致/差不多"、落到可见像素。
3. **分歧裁决**：逐格比对，不一致回到该帧（必要时拉 A2 疑点附近全帧）重看，取更严一方或升级。
4. 报告留两遍结论 + 分歧裁决，可追溯。

## 10. frame_audit.py CLI 契约（详到 Codex 可实现兼容版）

```
python frame_audit.py PROMPT集.md --clip N --video CLIP.mp4 [--video ...] \
       [--anchor-pack 锚点包.md] [--ref-dir 参考图目录] \
       [--scene-threshold 0.3] [--project-dir DIR] [--out-dir DIR]
```
行为（确定性，永不判分）：
1. **解析 Prompt 集**：复用 `xiaoyunque_phase8_runner.py` 解析（`CLIP_SECTION_RE`/`SHOT_TIME_RE`/`@图片N`），取该 Clip 分镜时段、镜头五段、朝向/站位/道具原话、引用编号。**（Codex 现工具缺此步，是最大差距。）**
2. **ffprobe 硬核对**（§5）→ 写 JSON + 报告确定性块。
3. **A1 全帧落盘** `frames\`（标准流程；失败 → 标"审片不完整、不可 PASS"，非视频 FAIL_BLOCKED）。
4. **A2 差分路由**：相邻帧 MAD/1−SSIM 或场景突变打分 → 运动曲线 + 三类疑点帧清单 + 真实切点位置，写 JSON。
5. **A3 分镜锚定取证**：每分镜入/中/出 + 切点对(本机拼) + 疑点帧，全分辨率拉出；另产低 fps 导航缩略图。
6. **生成工作表**：每分镜 Prompt 原话配到其取证帧组，判定列留空（§11）。
7. 退出：完成即 0；缺视频/缺分镜/ffmpeg 不可用 → 非 0 + 原因。**永不输出"对得上/对不上"。**

Windows 稳定性沿用：bytes 多编码容错（utf-8-sig/utf-8/gb18030）、不依赖默认编码、长中文路径加引号。

## 16. round-3 回 Codex（来自 Clip2 实跑 + 用户纠错）

> 背景：拿重返那年盛夏 Clip2 真跑了完整闸10（Part A 共享工具 → Part B Claude 主判 → Part C general-purpose 子代理对抗盲审 → 逐项裁决）。跑出 **4 个共享工具改进点 + 1 条判定协议升级**。
> **边界**：工具改动由 Codex 在共享 `tools\video-frame-audit\frame_audit.py` 实现（两树受益）——Claude 不擅自改你写的共享工具，只提建议。

### 16.A 共享工具 frame_audit.py 四个改进点

1. **确定性硬失败要显著、别被 `AUDIT_READY` 盖过**
   现状：`preflight`（全帧/差分/分镜映射）决定 `status`；但 `deterministic_checks` 的时长/比例哪怕命中 `RISK_OR_FAIL_REVIEW`，`status` 仍是 `AUDIT_READY`，硬失败只躺在子字段。
   建议：manifest 顶层加 `deterministic_block`(bool)，或 status 增 `AUDIT_READY_WITH_DETERMINISTIC_RISK`；报告头部拉成显著行。语义=截断/比例错是机器能定的 FAIL_BLOCKED 候选，必须显著，视觉层按 FAIL_BLOCKED 处理（除非显式接受测试风险）。

2. **比例容差分档（test vs production）**
   现状：`--ratio-tolerance` 默认 0.03；Clip2 864×496=54:31 离 16:9 差 2.02% 被判干净 PASS。
   建议：测试 480p 档可留 3%，**成片/生产档收紧**（如 ≤1%）；或对"在容差内但非干净比例"出 RISK 注记，别静默 PASS。

3. **scdet 阈值量纲确认**
   现状：`--scene-threshold` 默认 0.3；ffmpeg `scdet` 的 `lavfi.scd.score` 一般是 0–100（百分比）。
   建议：确认 0.3 是按 0–100 还是 0–1 标定；若 0–100，0.3 极敏感（静态戏 23 帧、动作戏会触顶 `flags[:50]`），考虑按片自适应或文档写清量纲，避免不同片灵敏度跳变。

4. **boundary 真切点定位改"邻域取最大"**（实跑暴露）
   现状：`scene_summary` 对每个声明边界取"离 `declared_time` 最近的尖峰"。Clip2 真切点在 3.71s（score 19.58），但 4.0s 处一个小 blip(0.453) 距声明 4.0s 更近被选成切点，**漏掉了数量级更大的真切点**。
   建议：改为"在 `[declared−tol, declared+tol]` 邻域内取 **score 最大** 的尖峰"作真实切点；邻域内无显著尖峰再回退最近。这样 3.71s/19.58 才会被正确认作真切点（并据此算早切量）。

### 16.B 判定协议升级：参照物锚定法（两树审片文档都加）

**缘起（关乎闸10 边界）**：Clip2 分镜2 脚本要"陈默面向窗、视线越过镜头望向窗外的光"。实际渲染窗在他**身后**、他**面向镜头**抬头看向上方/镜头前方、**没望向窗**。我方主判与**对抗盲审双双判 PASS**——两个模型都凭"抬头+被窗光打亮=望向窗光"模式匹配，没核对窗在身后。是**用户人工**抓出来的。Clip2 已改判 **FAIL_BLOCKED**（分镜2 朝向/视线）。

**结论三条**：
- **判朝向/站位锚参照物，不靠"被什么照亮"**。四步：①选参照物（脚本点名的窗/门/桌/床/楼梯/货架/车/产品/具名角色；没点名取画面固定地标；**光影不能当**）②定其画面方位（画左/右/深处/正后方/画外）③判人物相对它 **正对/背对/侧对(左·右)/俯/仰** ④对脚本。🔴 **被参照物的光打亮 ≠ 正对它；在身后=背对。** 套 Clip2：参照物=窗→正后方→人物背对窗→脚本要正对→FAIL。一并强化站位（相对地标左右前后）与跨镜越轴（相邻镜头相对**同一参照物**左右翻转无机位解释=越轴 FAIL）。
- **对抗复审不是万能的**：两个模型会共享几何盲点（轴外/身后的视线目标尤甚），对抗盲审没纠偏。**人工抽查这一环不能省**，写进规程。
- （可选）**工具可辅助**：frame_audit.py 解析镜头/朝向句时，把脚本点名的参照物抽出来填进报告模板"参照物"槽，提示视觉层去核对其方位与正/背/侧对——**工具只抽取不判定**。

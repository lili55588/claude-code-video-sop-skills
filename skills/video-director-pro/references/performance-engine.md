# 表演引擎（performance-engine）· Phase5 → Phase8 编排协议

> **定位**：把「角色台词与承重反应的表演真实性」做成 Phase5(动机设计) → Phase8(可见可听翻译) 的编排协议（单人 / 双人多人同框 / 过肩对话 / 具名对手反应均适用，见 §0）。**本文件是唯一真相源**——`phase5-storyboard.md` / `phase8-video-prompt.md` / `self-check-gates.md` / `SKILL.md` 只留入口+字段+红线链接,规则不在别处复制(防两处漂移)。
> **不是第四本表演词库**：表演细则仍归三本手册(`micro-expression.md` 脸/眼、`voice-control.md` 声音、`action-naturalness.md` 身体);本协议只负责「动机 → 分派 → 落可见可听」的编排与责任分流。**三本手册本轮不改内容**;若实现时发现必须新增底层规则,先回双根对齐、不单边追加。
> 来源：2026-06-22 Seedance audition 提示词的表演内核 + 双引擎「6 接缝」裁定与对齐(`给Claude_表演引擎_6接缝_Codex裁定`、`给Claude_Codex侧表演引擎落地改动清单`)。

## 0 · 适用与红线
- **适用**：有台词、有潜台词或情绪承重的表演 beat,**包括单人镜头、双人/多人同框、过肩对话及具名对手反应**。发动机**按每个承重说话回合或焦点反应角色分别设计**;不为不承重的背景人物建立发动机。纯信息台词标 P0、不加表演技巧。
- **来源契约(铁律)**：发动机所有字段必须**追溯已确认剧本 / 人物关系 / 当前情境**(Phase3 剧本、Phase4 人物、Phase5 情境)。**禁止从角色图自行推断**目标/动机/社会地位/创伤/潜台词——audition 那套「从图推角色」只适用于无剧本试镜,本流程有完整剧本。
- **self-tape 边界**：试镜/表演校准是独立可选模块(见 §7),**不进剧情 Clip**、不覆盖镜头/场景/剧情/关系。

## 1 · 表演等级 P0/P1/P2（绑剧本戏剧功能）
- **P2** = 剧本明确的：冲突升级 / 隐瞒被触碰或揭露 / 告白 / 揭示 / 权力翻转 / 不可逆选择 / 关系转折。**必带 剧本依据+转折触发,否则不得标 P2**。
- **P1** = 有关系压力或潜台词,但本镜**不发生**剧情转折。
- **P0** = 纯信息传递 / 流程性回应 / 不承重对白。
- **防通胀(定性,不设硬阈值——10s Clip 可含动作+反应两个合法转折)**：
  - 默认一个 Clip 一个**主导 P2 中心**;
  - 连续多 P2 必须各对应**不同**已确认 beat,并写清前一峰值释放 / 向下一 beat 交接;
  - 同一 Clip 所有台词都标 P2 → **强制复核**(不自动判死);
  - 剧本明确转折标成 P0、或普通信息句无依据标成 P2 = **等级错配,回 Phase5 修正**。

## 2 · Phase5.1 表演发动机字段（动机上游）
挂 Phase5.1、位于 `action-naturalness` **动作链设计之前**作动机上游;**不接管**角色站位/机位/道具承载面/动作路径(仍归既有模块)。
- **P0**：`表演等级=P0`,其余标 `NA(纯信息传递)`。
- **P1 必填**：剧本依据、目标、策略、潜台词、结束状态;障碍/转折触发/权力关系 按实际填或 `NA`。
- **P2 全填(八字段)**：剧本依据、目标、障碍、策略、潜台词、转折触发、权力关系、结束状态。
> 任何字段不得从角色图推断剧本事实。

## 3 · Phase8 翻译闸（导演术语执行化·编排层）
台词镜头写 Phase8 正文前,按固定顺序把发动机翻成**可见/可听指令**,按类**分派**三本手册(本闸只编排、不自带词库)：
```
剧本事实 → 目标/障碍/策略/潜台词 → 三力声音配方(内部) → 可听声音过程 → 可见表情/视线/身体 → 台词后反应与结束状态
```
- 比喻 / 表情 / 视线 → `micro-expression.md`(比喻置换、视线锚点、五查)
- 声音与台词过程 → `voice-control.md`(基础音色 + 本句声音过程)
- 身体 / 动作链 / 多人回合 → `action-naturalness.md`
- 潜台词 → 按输出类型分派到上三本(犹豫/回避/停顿 → 声音+表情;反应余波 → 动作+视线)
🔴 **Phase8 正文禁残留内部标签**：objective/obstacle/tactic/subtext/潜台词/转折触发/三力配方/Voice Trigger 等只是**中间表示**,正文只出现**可见动作 + 可听声音**(否则重蹈「描述果」)。

## 4 · 三力声音配方 → 本句声音过程（候选表,非映射）
三力配方(如 `cold, afraid, calculated`)是**内部中间表示**,**只生成「本句声音过程」、叠加在 Phase4 基础音色上**(基础音色仍是唯一跨 Clip 声音身份源、不被改写)。**禁一对一固定映射**(grief ≠ 机械颤音、fear ≠ 机械口吃);翻译器结合策略/标点/时长/等级,从候选里选**最少必要项**：
```
控制/克制 → 音量稳定、停顿受控、尾音收住
权威      → 咬字清楚、重音明确、句尾完整
恐惧      → 起音稍迟、气息变浅、某个重音不稳
悲伤      → 音量放低、呼气变长、尾音下沉或短暂卡住
试探      → 关键词前停顿、音量略轻、尾音保留
算计      → 节奏衡量、停顿精准、辅音偏利落
温柔      → 起音轻、音量柔和、语速略慢
```
最终写进 Phase8 的是「本句声音过程」(音量/语速/气息/停顿/重音/尾音),不是三个抽象形容词。

## 5 · 选择性触发（防套路：口吃/破音/低吼非每句必填）
| 等级 | 声音 | 表情/身体 | 特殊声音事件 |
|---|---|---|---|
| **P0** | 基础音色+自然尾音 | 正常视线与呼吸 | 不用 |
| **P1** | 一次停顿/重音/音量变化 | 一个微动作链 | 通常不用 |
| **P2** | 明确起势/峰值/收束 | 反应+转折+余波 | **最多一个** |
每句最多：一个明确视线目标 + 一个连续微动作链 + 一个声音过程 + (P2 才)一个特殊声音事件 + 一个结束/余波状态。

## 6 · 时长（并入闸4,不另立平行闸）
```
说话时长   = 台词字数(含标点) ÷ 5      (向上取整)
表演段时长 = 开口前非语言反应 + 说话时长 + 说完后余波   (顺序相加)
镜头时长   = ceil( max( 表演段时长, 其他动作时间轴, 信息扫描时长 ) )
```
- 能与说话**同时**发生的微动作**不重复加时**;明确发生在说话**前后**的反应**顺序计入**。
- P0 通常不单列前后反应;P1 最多一个短反应节点;P2 可有开口前 + 说完后两个承重节点。
- 装不下：先删不承重表演装饰;承重转折仍装不下 → 回 Phase5 延长或拆镜,Phase8 不硬塞。

## 6.5 · 镜头适配（景别决定表演细节层次）
- 远景/全景：体态/停步/肩颈/手部——**不写眼睑细节**。
- 中景：头部方向/视线/姿态/权力距离。
- 近景/特写：眼睑/嘴角/咬肌/喉部/呼吸。
- audition 的「画外读词者」在剧情 Clip **必须换成具名对手或画外声源**(不留试镜占位)。

## 7 · self-tape 角色表演校准（独立可选模块）
适用：新角色表演测试 / 反复声音假眼神飘 / 正式生成前测声线与表演区间。**不进剧情 Clip**。
产物经**用户人工择优确认**后写入 `{项目名}_角色表演基线卡.md`,记录：用户确认的基础声线参数 / 可用 + 不适用表演区间 / 稳定有效的停顿重音尾音气息 / 稳定视线微动作习惯 / 易过演失败模式 / 对应测试 Clip 版本与确认状态。
**写回规则**：用户确认后,基础声线**人工**写回 Phase4;表演区间/有效习惯/禁用模式供 Phase5 发动机引用。**不自动覆盖 Phase4/5**(模型生成的声音动作可能只是某次种子/提供方偏好,非角色事实)。基线卡不改视觉身份/剧情/机位/场景/关系。若实际音频作下游参考,**须按普通音频参考正式登记 + 排序 + 绑定**(闸5)。

## 8 · 验收（核心挂常规 Phase5/8 闸,不挂可选的 Gate9）
> 🔴 表演引擎**不依赖 Phase8.5**：即使不做宫格锚点也必须生效。核心验收挂 **闸6(连贯/音色) + 闸7(Prompt 结构/微表情五查/声音过程)**;**闸9 只在启用 Phase8.5 时追加**「锚点帧 vs Phase5 已确认表演起止/视线/身体结束状态 兼容」检查。**不得让「未启用 Phase8.5」成为跳过表演验收的路径。**

逐项(挂闸6/7)：
- 发动机每个 P1/P2 字段**可追溯已确认剧本**,无一从角色图推断;
- P0/P1/P2 与剧本戏剧功能匹配;**P2 必有 剧本依据+转折触发**;全台词 P2 触发复核;
- P1 只填轻字段、P2 才全填;站位/机位/道具/动作路径仍归既有模块;
- 时长：`镜头=ceil(max(表演段,其他动作,信息扫描))`,表演段含开口前+说话+余波,并行微动作不重复加时;
- 三力配方只影响本句声音过程;Phase4 基础音色仍唯一跨 Clip 声音源;无 grief=颤音等固定映射;
- P0 不强塞技巧;P2 每句 ≤1 特殊声音事件;口吃/破音/低吼非模板必填;
- Phase8 每个输出均为**可见/可听指令**,符合景别/视线几何/时间预算;**不改台词原文、不新增剧本不支持的转折**;
- **Phase8 正文不残留发动机内部标签**(机检 + 语义复核,见下);
- self-tape 基线卡须人工确认才写回;真实音频按普通参考登记排序绑定;
- **Phase5 + Phase8 同批落地,任一入口缺失则整套不启用**。

### 确定性 vs 语义,分边界（防「validator PASS 覆盖错觉」）
- **可机检** → `validate_phase8_prompt.py --forbid-performance-internal-labels`：正则**只抓发动机专有「字段名+冒号」标签**(`Objective:`/`Obstacle:`/`Tactic:`/`Subtext:`/`Voice Trigger A|B|C:`/`表演目标：`/`表演等级：`/`剧本依据：`/`潜台词：`/`转折触发：`/`三力配方：`);**不抓普通词**(目标/障碍/策略/结束状态/权力关系——都是合法镜头/剧情正文词,"视线目标"不误伤;实测"结束状态"曾误伤已剔除)。参数边界记 `validation-contracts.md`。
- **须语义检查(不可伪装成正则已证)** → P2 是否真对应剧本 beat、等级是否匹配、声音事件数量是否合理、字段是否真追溯剧本——由 LLM 跨 Phase 自检承担,**不能只写「validator PASS」当作已覆盖**。

# 9 道模型无关自检闸 + 闸10 成片视觉验收（回报「已完成」前自己跑）

> 这是即梦工作流硬规则的「验收闸」，由**你自己心算/对照执行**。每到一个阶段产物落盘、准备回报「已完成」前，跑对应的闸，把 PASS/FAIL 结果简述给用户；有 FAIL 先自修再交付。
> 对应关系：闸 1 贯穿全程；闸 2 在 P4/P7；**闸 2b（素材图质检）在 P7 回贴后**；**闸 2c（风格卡一致）在 P4/P7、仅 Phase1 产出过风格卡时**；闸 3/4 在 P5/P8；闸 5 在 P8；闸 6 在 P5.4/P8；闸 7 在 P8；**闸 7b（X-Tech 条件闸）在 P8、仅启用 X-Tech 路线时**；**闸 8（场景人口）在 P4/P5/P8**；**闸 9（片段视觉锚点包）在 P8.5、仅启用关键帧宫格锚点时**；**闸 10（成片逐帧视觉验收）在 P9、每个生成 Clip 进剪辑前，首个模型相关/视觉闸**。
> **Phase8 双层兜底**：闸 3/4/5/7 里**能正则确定**的部分（结构/时段连贯/逐任务引用/镜头·音色/全局要求段）已由确定性脚本 `scripts/validate_phase8_prompt.py` 机检——P8 落盘后**必跑该脚本且 `RESULT: PASS`**，再由你补判闸 1/2/6 与需语义判断的部分（用法见 `validation-contracts.md`）。

## 闸 1 · phase_gate（阶段前置闸）
**职责**：未满足前置/确认就不进下一 Phase。
**检查项**：P3 剧本已确认 → 才进 P4；P4 素材清单已确认 → 才进 P5；P5 最终 Clip 表 + 连贯性校验报告已确认 → 才进 P6；P6 比例已确认 → 才进 P7；P7 全量素材已确认**且全部过闸 2b 逐图质检** → 才进 P8；P8 全部片段已确认 → 才进 P9。
**FAIL 例**：`素材清单未确认即进入分镜设计（Phase5），阻断`。

## 闸 2 · asset_completeness（素材完整性）
**职责**：剧本/故事板/Prompt 中出现的实体必须都在素材清单里登记。
**检查项**：所有角色、关键道具、重要场景、营销产品均在清单（素材分 角色/物品/产品/场景 四类，营销强制单列产品）；**VFX 视觉机制 / 状态规则已结构化登记**（可选第五类，不能只散在分镜文字）；所有变体已登记且**主体=变体列表第一项**；有声主体均有音色描述（固定格式 `性别_年龄_语调_音高_语速_口音无`、多主体≥4 字段差异；**主要有台词角色建议含扩展字段** 声线质感/情绪底色/发声习惯，见 `voice-control.md`）；`主体来源` 三态合法（用户参考主体 / 用户普通参考图 / 待生成主体）。
**FAIL 例**：`故事板出现"织造局管事房"，但素材清单缺该场景`、`残影"冷灰蓝/暖橙红"只在分镜文字出现，未登记为 VFX视觉机制`。

## 闸 2b · reference_asset_qc（素材图质检，P7 回贴后）
**职责**：用户回贴的每张参考素材图，登记"已确认"前必须逐图核验（清单见 `phase7-refgen.md`「回贴素材图 QC 闭环」），不凭用户口头满意放行。
**检查项**：每张图都有状态 `PASS / REGENERATED_PASS / ACCEPTED_WITH_RISK / FAIL_BLOCKED`；`ACCEPTED_WITH_RISK` 必须记录具体风险点；身份一致（同角色多图不跑脸）、角色身份板按 `phase7-board-templates.md` 完整契约（必需分区齐全、主视图不裁脸）、产品文字/logo 一字不差无乱码、场景多视角设计板各格空镜无人物且无文字水印/各格同一空间一致（左大格 45-55%、已声明空间任务、无身体局部/动作道具/额外生物）、变体只改声明点、无五官/手部崩坏；质检结果 + 上传顺序已写入 `{项目名}_参考素材质检报告.md`。
**阻断规则**：存在任一 `FAIL_BLOCKED`、或非 PASS 且未记录接受风险 → **不得进 Phase8**。
**FAIL 例**：`R03_小晚身份板 侧视/背视发型与全身主视图不一致（跑脸），未重生成即登记"已确认"`、`R05 产品瓶身"本草集"印成"本革集"，未标 FAIL_BLOCKED`、`R07 场景设计板某格混入路人/某格与主布局非同一空间，未标 FAIL_BLOCKED`。
**VVK 增强（挂视觉验收内核 `visual-verification-kernel.md`，本闸 = VVK 在 P7 的实例）**：在原逐图契约上加——① **三层**：Part A 图片确定性证据（硬=分辨率/比例/文件；软=OCR文字logo/人脸数/水印/感知哈希，**只提示需视觉确认、不自动判死**）→ Part B 主判 → Part C 对抗盲审（主角身份/产品文字/核心场景/多视角必做）；② **确定性按 issue_class**（状态只用四态）：`EVIDENCE_GAP`=`AUDIT_INCOMPLETE`（修证据不改图）、`TECH_SPEC_BLOCKED`=`FAIL_BLOCKED` 容器层先（重导/裁切不伤主体）、`CONTENT_FAIL`=`FAIL_BLOCKED` 重生成；③ **OCR 二钥**：素材清单标"需一字不差"的字（产品名/logo/招牌/剧情字）才走 要求+OCR高置信+视觉确认→FAIL，incidental 水印/乱码可无损裁；④ **锁定状态** `LOCKED_PASS / LOCKED_WITH_RISK(风险注入下游) / REJECTED / AUDIT_INCOMPLETE`——**只有锁定的图能成为下游 P8.5/P9 参考**，REJECTED/INCOMPLETE 禁引用；⑤ **权威锚定链 + 依赖记录**：每素材记下游引用清单，改一张图沿链 cascade 复检引用它的宫格/Clip/成片；⑥ 返修写**区域坐标**（命名区+可选 bbox）。多视角锚主视图、场景锚布局、变体锚基底。

## 闸 2c · style_card（风格卡一致性，P4/P7，仅当 Phase1 产出过风格卡）
**职责**：风格卡存在时，下游必须按它复用、风格名不漂移、参考用途不串线。
**检查项**：
- 风格卡 `用户确认状态`=已确认 → **Phase7 必须走 A 分支复用**（不得无视风格卡又走 B 分支重新反推一遍），且 Phase7 产出的**风格名 = 风格卡风格名**（一字不漂移）；状态为 `待确认`/`已要求重做` → **不得进 A 分支**，先回 Phase1 把风格卡确认掉。
- 无风格卡（创意阶段没上传风格图）→ Phase7 走 B 分支反推即合规，不报错（避免误判）。
- 风格卡 `是否兼作场景参考`=否 → Phase4 **不得**把该风格图登记为任何角色/场景/产品参考；=`是，指定为场景X/角色X/产品X` → Phase4 **只**登记为指定的那一类那一个，未被其他类误登记。
**FAIL 例**：`风格卡已确认，Phase7 却重新反推画风、风格名从"日式赛璐珞动画"漂成"日系清新插画"`、`风格卡"是否兼作场景参考=否"，Phase4 却把它登记成女主的用户普通参考图（串线）`、`风格卡仍是"待确认"，Phase7 就进 A 分支复用了`。

## 闸 3 · shot_coverage（镜头覆盖）
**职责**：Shot 编号连续无遗漏无重复；Clip 覆盖所有 Shot；Prompt 集覆盖所有 Clip。
**FAIL 例**：`故事板含 Shot 1–38；Prompt 集仅覆盖 Shot 1–26、31–38；缺失 Shot 27–30 → 报错并要求补对应 Clip，禁止放行`。

## 闸 4 · clip_duration（时长约束，已对齐原生）
**职责**：单镜与 Clip 时长合规、各级总时长自洽。
**检查项**：每个 Shot 为**整数秒**且 ≤10s（小数即不合规，计算时向上取整）；每个 Clip ∈ [4, 10]s（装满优先、累加 8~10s 封箱；时长上限为用户实测覆写，官方 15s 已收紧为 10s）；单 Clip 子镜头 1~5 个；Clip 总时长 = 所含 Shot 计算时长之和（整数）；Prompt 标注时长 = Clip 总时长；全片总时长与预估误差 ≤ ±10%；有台词的镜头：**计算时长 = ceil(max(表演段时长, 其他动作时间轴, 信息扫描时长))**，其中 `表演段时长 = 开口前反应 + 说话时长 + 余波`（顺序相加、并行微动作不重复加时），自然 ≥ 说话时长（字数含标点 ÷5，向上取整）；表演计时见 `performance-engine.md` §6。标注「+1s 重叠缓冲」的 Shot 以 5.3 回写后的计时表为准（缓冲规则见 phase5.3）。
**FAIL 例**：`Clip 5 总时长 11s > 10s，需拆分`、`Shot 12 标 2.5s 非整数秒`、`Clip 3 含 7 个子分镜 > 5`、`Shot 8 台词14字仅给2s（14÷5=2.8 → 需≥3s）`。

## 闸 5 · reference_binding（引用绑定）
**职责**：Prompt 里出现的实体都正确带引用，且与素材顺序对齐。
**检查项**：Clip 中出现的每个角色/道具/场景都显式引用；用户参考主体用 `<subject>主体名</subject>`、普通参考用 `@图片N/@视频N/@音频N`（**待生成主体的标准图按普通参考处理，也用 `@图片N`**，判定只看有无 subject 标签）；`N` 从 1 起且与「参考素材顺序」严格一一对应、不跳号不错序；场景名与素材清单一致；正文禁用代称（他/她/女主/男主/两人/一行人）。
**FAIL 例**：`Clip3 提到"张嬷嬷"但无引用`、`@图片2 在参考素材顺序中无对应资源`、`正文出现代称"两人"`。

## 闸 6 · continuity（连贯性，对应 Phase5.4 六步校验）
**职责**：跨 Clip 一致性机检。
**检查项**：角色服装/发型/伤痕/状态延续（状态变化后续镜持续标注直到解除）；道具位置/数量/状态延续，隐式道具已入「物品补充设计清单」；场景时间/天气/光影延续；角色站位每镜写清；相邻 Clip 动作不重复（5.3 标注「重叠缓冲」的边界除外：复现须与前 Clip 结尾动作状态一致、缓冲后不再二次重复）、运镜有交替、空间瞬移补过渡；背景群像不凭空消失；**焦点角色视线目标与情绪状态跨 Clip 延续或可见释放**（情绪残留四类：哭后眼眶泛红泪痕、压怒后咬肌紧绷手攥拳、震惊后眨眼延迟、释然后肩落长呼气——延续到剧情写出可见释放；上一 Clip 锁定对方的视线，下一 Clip 要么延续要么写出移开动作）；**同一角色基础音色跨 Clip/任务一字不差**（各 Clip 独立并行生成，基础音色漂移=换人声；仅"本句声音过程"随剧情递进，且不无过程回跳，见 `voice-control.md` 第四节）。
**FAIL 例**：`Shot34 后苏晚右肩破损渗血，Clip8 未延续该状态`。

## 闸 7 · video_prompt_structure（Prompt 结构）
**职责**：校验每条视频 Prompt 结构完整（对齐 Phase8 官方 8-A）。
**检查项**：
① 开头一句全局风格；
② `生成一个由以下N个分镜组成的视频：`；
③ 每个分镜：**整数时间段** `0-Xs`（禁括号/小数）+ 镜头五段写法 `镜头：景别，画面落点，拍摄距离，从哪拍摄，运镜方式。` + 动作 + 台词 + 背景；
③b 🔴 **每个分镜正文都显式嵌入场景图引用 `@图片N`**（场景为用户参考主体时用 `<subject>场景名</subject>`；**纯文字场景名不能替代引用**）——**含中间近景/特写分镜**，禁靠上下文隐含（防场景漂移，即梦官方必填项；用 `validate_phase8_prompt.py --scene-image N` 机检兜底，跨场景 Clip 可重复传多个编号、并由本闸语义复核"各分镜嵌的是不是自己所在场景"）；
④ 有台词则**音色紧跟台词后**（`…说：「台词」音色：…`），且音色按**声音结构**写（见 `voice-control.md`）：基础音色与素材清单一字不差 + 本句声音过程（重音/停顿/尾音/音量/气息至少其一），**禁止裸情绪词音色**（只写"愤怒地/悲伤地"无声音过程）；多句台词情绪有变化时逐分镜标声音递进；多角色不共用音色描述；
⑤ 末尾全局要求段（按需补「禁止生成任何台词/旁白字幕，禁止生成背景音乐。」「配音声音不要戛然而止，要留自然的尾音。」「语言：中文。」）；
⑥ 时长策略正确（全片 ≤10s 单任务）；
⑦ 无电影名/导演名/名人名（出现则替换为中性摄影描述）；
⑧ **微表情与视线五查**（防僵硬/防眼神飘，见 `micro-expression.md`）：无裸奔大情绪词（嚎啕大哭/放声大笑/极度震惊——须已拆成 Start→Transition→End 过程或微量表达；**营销"夸张反应"镜头豁免强度、但仍须有过程**）；无会被字面执行的比喻（眼睛发光/亮了、脸色阴沉、目光如刀——须已物理置换，真实光源反射除外）；无"全程直视镜头"的静态描述（直视须有抬眼过程）；**每个可见正常视力的焦点角色有明确视线目标或可见"不看"理由**（对话戏说话者/听者视线匹配场面几何，过肩镜头写清视线越过谁落在谁脸上）；**失焦仅限剧情明示状态**（失明/恍惚/重击/醉酒/解离）且有重新聚焦时刻。
⑨ **台词镜头表演引擎（P1/P2，详见 `performance-engine.md` §8）**：发动机字段可追溯已确认剧本（无一从角色图推断）；P0/P1/P2 与剧本 beat 匹配、P2 必带剧本依据+转折触发、全台词 P2 触发复核；三力配方只产「本句声音过程」叠基础音色（基础音色仍唯一跨 Clip 源）；P0 不强塞技巧、P2 每句 ≤1 特殊声音事件、口吃/破音非必填；正文不残留 objective/obstacle/tactic/subtext 等内部标签（机检 `--forbid-performance-internal-labels` + 语义复核）；表演段时长并入计时；微动作符景别、视线符空间；不改台词原文、不新增剧本不支持的转折。**本验收挂闸6/7、不依赖 Phase8.5**；启用 Phase8.5 时闸9 仅追加「锚点帧 vs 表演起止状态」兼容检查。
**FAIL 例**：`分镜2 只引用了苏晚@图片2、漏嵌场景@图片1（场景漂移）`、`Clip2 缺末尾全局要求段`、`台词缺音色描述`、`台词后音色仅写"她悲伤地说"，缺声音过程（音量/语速/停顿/尾音）`、`Clip3 苏晚基础音色与 Clip1 不一致（年龄24→22）`、`出现"诺兰风格"需替换为中性描述`、`分镜3 写"她眼睛亮了起来"（会字面生成发光），须置换为"视线骤然聚焦，眼睛微微睁大"`。

## 闸 7b · x_tech_oak_koda（X-Tech 条件闸，仅当 `x-tech-oak-koda-workflow.md` 已加载/项目用 X-Tech·OAK·Koda·triptych·infer-between·Extend 路线时跑）
**职责**：X-Tech 路线的 Clip 按其专属规则过闸——不被 8-A-only 结构断言误伤，也不放过 infer-between 的事实漂移。非 X-Tech 项目整闸跳过。
**检查项**：
- **候选探针只提议**：Phase5/Phase8 常开轻量探针可以在没有触发词时标出 X-Tech 候选，但它只做路线建议，不能自动执行 X-Tech 路线。
- **批量提议 + 用户确认**：明确候选必须一次性列出候选 Clip、建议 Route A-F 和一句理由；记录用户确认后，才允许加载/套用完整 X-Tech 路线。
- **未确认回严控**：被拒绝或未确认的候选 Clip 仍按标准严格 8-A 生成；没有高置信候选时不提 X-Tech。
- **单 prompt 不混松严**：每条 Clip 只一个 Route（标准 8-A 或 A/B/C/D/E/F）；无单 prompt 同时写「三联画 infer-between 脑补」与「逐拍 8-A/storyboard 严执行」。
- **Reference Authority Contract**：项目顶部有全局 Contract；换 Route 或换权威的 Clip 写了 `Contract override`；IDENTITY/STRUCTURE/STAGING/LOOK/ACTING 各且仅一个权威，无抢权（身份板与 triptych 不同时声明控 look）。
- **infer-between Clip**：已显式标 `TEXTURE-DRIVEN`（事实驱动/带货/因果动作 Clip 留严控 8-A 或 Route D/E）；用 `X-TECH INFER-BETWEEN CLIP` 专用块、非伪 8-A 多分镜；**FACT-LOCK 七项齐**（身份/服装/标志、Clip 级场景 `@图片N`、道具白名单、population、不可发明、1~3 关键 beat、放权范围），不齐则 infer-between 阻断、回退 Phase5 或 8-A；**它豁免逐分镜 `@图片N` 机检，但 FACT-LOCK 必含场景 ref + population**；**校验器现已机检本格式**（`validate_phase8_prompt.py --expected-xtech-infer-between N`：FACT-LOCK 七字段 + 不混 8-A + FACS/`AU\d+` 拦截，2026-06-27 三根对齐）——机检兜底 + 本闸人工语义（beat 是否真对应剧本、acting code 语义）+ Gate10 像素事实，三层把关（见 `x-tech-oak-koda-workflow.md` §5.7）。
- **Route F Extend**：串行依赖（先有 video1）、不与独立 Clip 批量并行；连续 take 连贯（地点可在 take 内变化）、非跨场景表演迁移。
- **FACS/IPA/Laban 不进正文**：内部编码层不出现在最终 Phase8 正文（现 validator 未机检这些词，靠本闸语义守）。
- **10s 默认**：X-Tech 的 15s 仅用户显式风险模式，未把 15s 当默认。
**阻断规则**：infer-between Clip 缺完整 FACT-LOCK、或单 prompt 混松严、或多 reference 抢同一维度 → **不放行该 Clip**。
**FAIL 例**：`某 Clip 标 infer-between 但 FACT-LOCK 没写场景 @图片N → 阻断`、`一条 prompt 里既写三联画脑补又逐格 storyboard 执行`、`身份板与 triptych 都声明控 look，未在 Contract 分清`、`infer-between Clip 拿普通 8-A 校验器 RESULT: PASS 当交付门`。

## 闸 8 · scene_population（场景人口，P4/P5/P8）
**职责**：防"参考图空镜 → 成片照空场景生成无人画面"（教室没学生/餐厅没食客/街道没行人/宴会没宾客——最高频穿帮）。场景参考图永远空镜（只锁空间），但成片人数必须独立声明并逐层贯通。
**检查项**：
- **P4**：每个场景条目都有 `场景人口` 字段（空镜无人 / 仅主要角色 / 主要角色+背景群像并注明身份+密度）；公共/营业场景（教室/餐厅/商场/街道/站台/宴会/办公室等）按常识时段标了背景群像、未误标空镜；同场景不同时段人口不同的随场景变体分别标。
- **P5.4**：标了"主要角色+背景群像"的场景，拆镜时背景群像已在第一个同场 Clip 建立、且**每个同场分镜都交代**（含特写/近景），不只首镜有、后镜消失。
- **P8**：每个分镜正文按 `场景人口` 显式写出画面人数——该有背景群像的场景每个分镜都写了"背景群像、景深虚化"，`空镜无人`写明无人；背景群像未误用 `@图片N`/未给全名台词/未写精修五官。
**FAIL 例**：`场景"高中教室·白天午后"剧情是上课，素材清单 场景人口 漏标、Phase8 各分镜只写课桌没写学生 → 即梦生成空教室`、`站台场景仅 Clip23 写了人流，Clip24-27 背景人群消失`、`把背景同学写成"@图片5 同学"当主体引用（应为无引用的环境群像）`。

## 闸 10 · generated_video_visual_audit（成片逐帧视觉验收，P9，每个生成 Clip 进剪辑前）
**职责**：工作流**第一道模型相关（视觉）闸**——成片渲染后逐帧核对真实像素是否对得上分镜 Prompt。闸 1–9 全是出片前文本闸，本闸是出片后像素闸。平台通用（即梦/小云雀/Seedance）。完整规程见 `phase9-frame-audit.md`。
**触发**：任一 Clip 成片到手后、回报该 Clip「已接受」/进剪辑交接前**自动跑**；含人物/复现道具/动作连贯/台词/视线/跨 Clip 关系的 Clip 全维度审，纯抽象/空场景 Clip 仍查场景/时长/镜头/artifact。共享工具 `C:\Users\Administrator\Desktop\tools\video-frame-audit\frame_audit.py`（解析 Prompt+全帧+差分+分镜锚定取证，永不判分）。
**前置完整性闸（先于四态）**：全帧落盘 + 差分 + 分镜映射齐 → 工具状态 `AUDIT_READY`（=证据齐、**未判定**，≠PASS）；未齐 → `AUDIT_INCOMPLETE`，**硬阻断**（禁 PASS、禁进剪辑、禁说 Clip 可用），先补审片，不得弱化为 WARN。`AUDIT_INCOMPLETE`≠`FAIL_BLOCKED`（前者修工具/重抽，后者改视频/回退）。
**检查项**：逐分镜核对——分镜对位（差分定位真实切点，偏差>0.5s FAIL）、人物朝向（**参照物锚定法**判正对/背对/侧对——先定脚本点名的视线锚点(窗/门/桌/人/物)在画面方位、再判人物相对它正/背/侧对；**被其光照亮≠正对它**，详 `phase9-frame-audit.md`）、站位漂移（相对参照物的左右前后）、道具漂移（承载面/归属手/开合）、跨镜一致（切点帧对无解释跳变）、身份不跑脸、动作节拍/口型/收尾pose、场景人口/光向/地标、锚点执行。
**确定性硬核对（机检，可定 FAIL_BLOCKED）**：时长 vs Prompt 标注（截断=FAIL_BLOCKED）、比例 vs Phase6（错=FAIL_BLOCKED 除非显式标测试可接受）。🔴 **工具把这些标 `RISK_OR_FAIL_REVIEW` 时，视觉层必须按 FAIL_BLOCKED 处理，不得被头部 `AUDIT_READY` 一行盖过**；比例容差成片档应收紧、不照搬测试档宽松值。
**反糊弄 + 对抗复审**：默认存疑、PASS 也要给正证据（帧@t+观察 vs 原话）、怀疑项先穷举、二元勾选表、看不清只降级不默认 PASS；主判后派独立子代理**盲审**同批取证帧（不给看主判结论），分歧逐格取更严一方。
**五态**：前置 `AUDIT_INCOMPLETE`（补证据）→ 质量四态 `PASS / ACCEPTED_WITH_RISK(写明风险) / FAIL_BLOCKED / REGENERATED_PASS`。
**阻断规则**：任一维度 `FAIL_BLOCKED` 或前置 `AUDIT_INCOMPLETE` 的 Clip → **不进剪辑**，走 **Gate10 返修决策层**（见 `phase9-frame-audit.md`：证据→内容先定稿→容器(交 video-auto-edit·不遮盖)→局部可删(承重四问)→改 Prompt 重生成(责任分流·重试预算 K)→接受风险；铁规则=剪辑只能真修复不遮盖、内容重生成先做容器批量后做；每次处理后重跑闸10）。未过本闸不得回报该 Clip「可用」。
**FAIL 例**：`Clip2 分镜2 Prompt 写"侧身面向窗"，成片全程正脸对镜头 → 朝向 FAIL_BLOCKED`、`成片 5s 但 Prompt 标 9s 截断，工具标 RISK_OR_FAIL_REVIEW 却被当 AUDIT_READY 放行 → 漏判`、`只抽 3 张预览帧就说没问题（未全帧）→ AUDIT_INCOMPLETE 硬阻断`。

## 返修闭环（用户/闸指出问题时，最小影响，不全流程重做）

1. **收集问题**：把反馈/FAIL 列成清单，定位到具体 Shot/Clip/素材。
2. **分类 → 定位 Phase**（用 `rules-and-fallback.md` 回退表）。
3. **修上游素材清单**（缺素材/缺 VFX 机制/缺状态规则 → 回 P4 补登记，标主体来源，必要时同步物品补充设计清单）。
4. **补生成参考图**（P7 对新增素材出生成提示词，图生图保持风格一致，用户确认）。
5. **补/改下游 Clip 与 Prompt**（P5 补分镜并合进 Clip，或 P8 改对应 Prompt）。
6. **重排编号**：Shot/Clip 编号重新连续化，更新「包含镜头」「参考素材顺序」等所有交叉引用。
7. **重跑全部闸**：覆盖、引用、时长、连贯性、结构、阶段闸全 PASS 后才输出修正版。

**返修红线**：禁止为消除穿帮私自改剧情；禁止新增/删除已确认素材而不更新清单；禁止只改下游不补上游；返修后必须重新过闸再交付。

# Phase 7：参考素材生成（story-ref-gen / ecom-ref-gen）— 4 步流程

## 核心定位
视觉一致性的**核心保障**。基于 Phase4 素材清单批量生成所有参考素材（角色/物品/产品/场景及变体），后续视频生成都以此为基准。**本 Skill 不重新做素材深化设计**，只读取并继承 Phase4【素材清单】+ Phase5.4【物品补充设计清单】的现成描述去生成。
> 类型适配：故事类走 story-ref-gen，电商/营销类走 ecom-ref-gen。

## 🔧 本机执行模式（手动跑即梦）
本机没有 `text2image / image2image`，所以本阶段**不真的出图**，而是按 4 步逐项**产出"参考素材生成提示词清单"**，让你拿去即梦 UI 跑。你跑完把图**回贴给我**——我**先逐图质检（见下方「回贴素材图 QC 闭环」）、全部 PASS 或明确接受风险后才登记"已确认"**并锁定「参考素材顺序」，不做"用户说满意就直接登记"。原生 CLI 的模型版本/绑定机制见文末附录（手动模式不必关心，接 CLI 时再用）。

## 前置准入铁律
- ✅ Phase6 全局比例已确认；最终素材清单已 100% 确认无增删改；所有素材主体来源、变体需求已明确。
## 跳步判定
清单所有素材主体来源都是「用户参考主体」、无「待生成主体」，且所有变体用户已提供 → 跳过进 Phase8。

## 步骤一：读取前置清单（继承，不重设计）
- 以【素材清单】为唯一设计依据，角色/物品/产品/场景主体与变体描述直接取自清单，**不得自行改写或重设计、不删改、不擅自新增**。
- 生成范围写死 = 清单 + 物品补充设计清单中**全部应生成素材**，不得遗漏、不得自行筛"核心素材"。
- 主体来源以清单 `主体来源`/`参考主体` 字段为准，**禁止仅凭正文猜测**。

## 步骤二：全局风格探索与确立（生成第一个场景的多视角设计板作风格图）
**风格图作用**：锁定**画风 / 造型语言 / 材质表现 / 笔触方式**——**只推荐"画风/艺术形式"，禁用色调/情绪/氛围词**（"温暖治愈""悬疑紧张""电影感写实"这类**不是**合法风格候选；要的是可区分画风）。
- 风格名统一写法：`参考xx图的画风，xx风格`（如「参考@图片1的画风，日式赛璐珞动画风格」）；禁含混堆叠名（"梦幻电影感复古插画风格"❌）。
- 风格候选参考池：日式赛璐珞动画 / 美式复古漫画 / 欧美3D卡通 / 欧美2D手绘 / 黏土定格 / 低多边形3D / 像素艺术 / 纸雕拼贴 / 国风工笔重彩 / 水彩手绘插画 / 油画厚涂 / 黑白木刻版画 / 扁平矢量海报 / 手作布偶定格。
- **禁用**：具体导演/工作室（诺兰/吉卜力）、IP/品牌（漫威/迪士尼）、艺术家（宫崎骏/今敏）。
- 风格图按**第一个场景的空场景多视角设计板**格式生成（16:9 横屏、板式需要不随全局比例）：左侧大格锁画风+主空间布局，其余小格给低机位/侧视/近景背景/前景遮挡等实用机位，各格属同一空间（地标/光线方向/地面纹理互相对应）；**全程无人物/无文字水印**；把第一个场景自带信息（地点+时间+内/外+天气）一起带进 prompt，外景必须体现雨幕/积水/反光等。**左侧大格必须清楚承载已选画风**，使这张设计板仍能作为后续所有素材的画风锚点。
- **风格来源先判一刀（单一决策点，先于下面四种来源——避免与 Phase1 风格卡形成"双入口"）**：
  - **A · 已有 Phase1「已确认」的风格卡**（风格卡 `用户确认状态`=已确认；`待确认`/`已要求重做` 的不算，需先回 Phase1 把风格卡确认掉再来）→ **直接取风格卡的风格名 + 全片视觉规则，跳过下面的画风总结/推荐，不重复反推**；**风格名严格沿用风格卡、一字不改写漂移**；风格图生成按风格卡 `是否兼作场景参考` 字段分两种走法：
    - `是否兼作场景参考＝否`（默认，纯风格参考）→ 用「风格卡画风 + 第一个场景描述」生成**第一个场景的多视角设计板**作风格图（同时就是第一个场景的场景图），表单确认。
    - `是否兼作场景参考＝是，指定为场景X` → 该上传图**就用作场景X的场景图**（用户实景上传图按原样登记、不强制做成设计板）：若场景X即第一个场景，**直接以该上传图作第一个场景的风格图/场景图、不再另生成**；若场景X非第一个场景，第一个场景的风格图仍按「风格卡画风 + 第一个场景描述」生成**多视角设计板**，而**场景X的场景图复用该上传图、步骤三不再为场景X重复生成**（避免重复生成/覆盖已指定的场景参考）。
    - `是否兼作场景参考＝是，指定为角色X / 产品X` → 风格图照常按「风格卡画风 + 第一个场景描述」生成；该上传图按 Phase4 登记给指定的角色X/产品X 作普通参考图，步骤三该主体严格参考它（不影响风格图本身）。
  - **B · 没有风格卡**（创意阶段没给风格图）→ 按下面四种来源处理；其中来源 1/2/3 需要的「总结画风」动作，用**五维反推框架现场反推**（镜头参数→布光→人物与场景→色彩与后期→复刻方案，完整定义见 `references/phase1-2-idea-outline.md`「风格参考反推」），产出风格名 + 全片视觉规则；来源 4 直接推荐画风。
    1. 上传角色/物品图（无场景图）→ 先按五维总结其画风，用该画风 + 第一个场景描述生成第一个场景的多视角设计板作风格图，表单确认；
    2. 上传场景图 → 按五维总结画风，以该上传场景图为风格图（用户实景图按原样登记、不强制做成设计板），确认；
    3. 仅给风格描述 → 按描述 + 第一个场景生成第一个场景的多视角设计板，确认；
    4. 啥都没给 → 推断题材 → 推荐 **4 种差异化画风**，并行给 4 张第一个场景的多视角设计板（每种画风各一张），用户选 1。
- 无论 A 还是 B，都守：反推/总结结果两用（喂给风格图 prompt 的画面规则部分 + 沉淀一句「全片视觉规则」供 Phase8 全片一致性约束引用）；风格名只收敛"画风/艺术形式"，色调/光影/情绪只进画面规则描述、不得混进风格名。
- 风格图既然基于第一个场景，**它同时就是该场景的场景图**，步骤三不再重复生成同一场景。
- 确认话术：「好的，已确认使用【XX风格】，这张作为后续所有素材的全局视觉风格基准。接下来依据【素材清单】和【物品补充设计清单】生成全部参考素材。」

## 步骤三：生成全部主体（不生成变体）
**优先级**：核心角色/产品 → 次要角色/重要道具 → 核心/次要场景。**单体素材 prompt 只描述该单体本身**，不带入其他角色/道具/场景/互动。

按主体来源分别处理：
| 主体来源 | 处理 |
|---|---|
| 用户参考主体（带 `<subject>`） | 直接复用，**不重复生成主体图、不补身份板**；后续引用写 `<subject>{主体名}</subject>` |
| 用户普通参考图（无 `<subject>`） | 这类不是"已提供主体"、不能跳过；**严格参考该图、按其素材类别生成对应标准资产**（角色→身份板 / 物品→单体图 / 产品→三视图 / 场景→多视角设计板）；prompt 写「严格参考」 |
| 待生成主体 | 基于已确认风格图生成**对应标准资产**（角色→身份板 / 物品→单体图 / 产品→三视图 / 场景→多视角设计板） |

**角色身份板 / 场景多视角设计板必须按 `references/phase7-board-templates.md` 的完整契约写 prompt**（角色逐字段填实完整 CHARACTER IDENTITY BOARD 模板；场景按完整设计板契约），**禁压成三视图或单张氛围图**。两个速记：
- 角色身份板：大幅全身主视图＋全身中性站姿＋背/侧视＋次姿态＋4~6 表情研究＋服装/道具特写＋剪影＋色卡，纯白底、版式分隔不裁脸；画风随风格图。
- 场景设计板：3~5 无文字分格，**左大格占 45%-55%** 锁主空间布局，余格给低机位/侧视/近景背景/前景遮挡；同一空间（地标/光向/材质/尺度/路径/出口一致），**必须声明空间任务（入口/过渡/阻碍/转折/开阔收束）、以本场最强空间特征开篇（反同质化）**，中景留表演区、后景可读，**禁人物/动物/身体局部/动作道具/文字/水印**。作该场景所有镜头统一参考、按机位取格。
**"重新生成"是 prompt 关键字**（待生成主体的角色/物品/场景必带，不可省）。
**角色身份板、场景多视角设计板用 16:9 横屏；产品三视图（主体+变体）固定 3:2 横屏**（参考资产，板式需要、不随全局比例；最终产品镜头才用全局比例）；角色/物品纯白背景（身份板亦纯白底）；**场景设计板各格空镜无人物、无文字水印**。

**标准 Prompt 示例（故事类）**：
- 待生成主体·角色：`参考@图片1（风格图）的画风，国风工笔重彩风格，重新生成一位18岁古代织娘：柳叶眉杏眼、梳双丫髻、蓝色粗布丫鬟服…，仅生成该角色单体本身。按 references/phase7-board-templates.md 的完整 CHARACTER IDENTITY BOARD 契约逐字段填实并出 16:9 身份板，仅呈现该角色、不出现其他角色或场景。`
- 用户普通参考图·角色：`严格参考 苏晚@图片2 作为最高优先级身份来源，保持人脸、发型、年龄感、体型比例、服装结构、永久配饰、配色和识别性细节不变，不重新设计角色。按 references/phase7-board-templates.md 的完整 CHARACTER IDENTITY BOARD 契约出 16:9 身份板。`
- 用户参考主体·角色：`严格参考<subject>苏晚</subject>的外观与主体特征，保持人脸、发型、体型完全一致，仅用于后续变体和组合生成，不重复生成该角色主体图。`
- 物品：`参考@图片1（风格图）的画风，国风工笔风格，重新生成一把竹制织机零件：…，仅生成该物品单体本身，纯白色背景，禁止生成人物。`
- 场景：`<本场最强空间特征开篇>。参考@图片1（风格图）的画风，国风工笔风格，重新生成织造局偏院机房的 16:9 空场景多视角设计板：3~5 无文字分格，左侧大格占 45%-55% 锁定主空间布局，其余小格给低机位、侧视、近景背景、前景遮挡等实用机位；所有分格属同一空间，地标/光源方向/材质/尺度/表演区/路径/出口一致。场景信息：织造局偏院机房（清晨/内），空间任务=过渡。中景留表演区、后景可读；若为外景必须明确呈现对应天气、空气状态、地面痕迹。禁止任何人物/动物/身体局部/动作道具/文字/水印。完整契约见 references/phase7-board-templates.md。`

## 步骤四：基于主体生成变体（主体确认后才做）
- **禁止用变体套变体**；每个变体都基于**同一个已确认主体来源**分别生成；变体输入必须含主体（步骤三主体图 或 用户参考主体），风格图只作辅助、不能成为唯一输入。
- prompt 必须写「**保持未声明改变的核心识别特征一致，仅按声明改 YY**」，且**变化点写具体**（"头发花白、面部皱纹增多、身形略微佝偻，其余不变"，不写"老年版"）；**声明要改的就改、没声明的不变**——童年/老年变体允许年龄、体型、年龄痕迹按声明变化，受伤变体允许伤处变化；**不要笼统写"保持年龄/体型一致"**，那会和童年/老年变体自相矛盾。
- **变体保留主体板结构（与主体同格式出板）**：角色变体延续身份板版式与各视图角色（panel roles：全身主视图＋背/侧＋表情研究＋色卡，只把声明的改动点贯穿到各视图）；场景变体延续 3-5 分格设计板布局（左大格＋实用机位格、同一空间、空间任务不变），只改声明的状态（天气/时间等）。产品变体不出板、保持三视图。
- 典型变体：角色（童年/老年/受伤/换装/伪装/非人化）；物品（开关/完整破损/包装前后/使用前后）；场景（白天夜晚/营业打烊/整洁凌乱/晴雨，保留地点时间内外天气）。
- 示例（角色变体）：`严格参考@图片(苏晚身份板)的外观与主体特征，延续@图片(风格图)的画风，生成苏晚的礼服变体，按身份板同样版式出 16:9 身份板（全身主视图＋背/侧＋表情研究＋色卡），保持人脸、发型、年龄感与各视图结构一致，仅将服装改为进宫礼服、头戴珠花贯穿各视图，其余不变，纯白色背景。`

## 回贴素材图 QC 闭环（登记"已确认"前必跑，对应自检闸 2b）

> 🔧 本闸挂**视觉验收内核 VVK**（见 `visual-verification-kernel.md`）：三层（确定性证据 + 视觉主判 + 对抗盲审）、确定性三级（`AUDIT_INCOMPLETE`/技术规格阻断/内容 `FAIL_BLOCKED`）+ 软信号（OCR/人脸数/水印/哈希）**只提示需视觉确认**、**OCR 二钥**、**锁定状态**（只有 `LOCKED_PASS`/带风险的图能进下游）、**权威锚定链 + 依赖记录**（改图沿链 cascade 复检下游）。下面的逐图契约是 P7 的检查矩阵实例。

用户把即梦跑出的图回贴后，**逐图对照【素材清单】+【物品补充设计清单】核验**，不是看个大概。我能直接看图，这一步是本机的硬职责，不外包给"用户觉得差不多"。

**逐图检查清单**：
- **通用**：角色身份板含必需分区（大幅全身主视图＋背/侧视＋面部/表情研究＋色卡，主视图从头到脚完整）；产品三视图=正/侧/俯；场景多视角设计板含主布局大格＋实用机位小格、各格属同一空间；无多余人物/水印/无关道具；画风与已锁定风格图一致；无真实品牌/IP/名人元素混入。
- **角色**：人脸、发型、体型、服装、鞋/足部与清单描述逐项一致；身份板主视图从头到脚完整不裁切、无裁脸，板内各视图（背/侧/表情研究）与主视图为同一人不漂移；**同一角色多张图身份一致（防"跑脸"）**。
- **产品**：品牌名/logo/型号/包装文字与「产品文字信息清单」**一字不差、无乱码不变形**；瓶型/盒型/材质/颜色一致。
- **物品**：单体出镜，无手持/摆放关系/人物。
- **场景**：多视角设计板各格均空镜无人物、无文字水印；各格属同一空间（地标/光线方向/草高/地面纹理互相对应）；中景留表演空区、后景可读；时间/天气/光线/空间结构/关键陈设与清单一致（外景天气痕迹必须可见）。
- **变体**：确实基于主体（身份/结构/文字未漂移），**只改了声明的变化点**，其余不变。
- **质量**：无五官/肢体/手指崩坏，无明显廉价感（对照上文「参考图 Prompt 质量增强」的规避清单）。

**每张图给一个状态**（写入质检表）：
| 状态 | 含义 |
|---|---|
| `PASS` | 逐项相符，可直接用 |
| `REGENERATED_PASS` | 首检不合格 → 我出**针对性修正提示词**（图生图、只改失败字段）→ 用户重跑回贴 → 复检通过 |
| `ACCEPTED_WITH_RISK` | 有瑕疵但用户明确接受，**必须记录具体风险点**（如"侧视图鞋型略不一致，正面镜头为主可接受"） |
| `FAIL_BLOCKED` | 关键不合格且未解决，**阻断 Phase8** |

**闭环规则**：
- 同一张图**修正 3 轮仍不过** → 标 `FAIL_BLOCKED`，停下来向用户给选项（换生成思路 / 明确接受风险 / 换素材来源），不死磕也不放行。
- 用户**不便回贴图片**（只口头说"跑完了都挺好"）→ 明确告知我没看到图、无法逐图质检，标 `ACCEPTED_WITH_RISK`（风险=未经逐图质检）后继续，**不冒充已质检**。
- 质检结果 + 即梦上传顺序写入 **`{项目名}_参考素材质检报告.md`**（与项目既有命名一致），格式：

```markdown
| ID | 素材 | 对应清单条目 | 检查要点 | 状态 | 处理/风险 | Phase8 角色 |
|---|---|---|---|---|---|---|
| R01 | 场景设计板_机房 | 场景·织造局偏院机房 | 多机位同空间/各格空镜无文字/清晨/光线/陈设/风格 | PASS | — | 任务1 @图片1 |
```
- **存在任一 `FAIL_BLOCKED`、或非 PASS 且未记录接受风险的图 → 不得进 Phase8**。

## 通用铁律
- **并行硬规则**（手动模式 = 一次把所有提示词给全让用户一批跑，不挤牙膏）：彼此独立的风格图/角色/物品/场景/变体一次性提交，不串行、不分批。
- **完整性铁律**：覆盖清单所有主体 + 所有变体，不遗漏、不自行筛选。
- **修改图铁律**：用户要改/重做素材，必须**图生图**基于现有图调整，禁文生图从零重做。
- **用户确认铁律**：所有主体 + 所有变体全生成完才统一确认，**禁止在主体与变体之间插入确认**。
- **素材成功铁律**：任一失败/缺图/资源不可用必须补生成；禁跳过、禁用文字描述代替参考图、禁带病进 Phase8。
- **角色身份板铁律**：普通参考图 + 待生成主体的单体角色都要生符合全局风格的角色身份板；参考主体默认复用不重生。
- **纯白背景**：角色/物品参考图纯白底（角色身份板亦纯白底）；**场景多视角设计板各格必空镜无人物、无文字水印**。
- **话术先行**：先输出引导话术再"提交"，不直接甩工具调用。⚠️ 禁止把素材资源 ID 展示给用户。

## 营销/电商分支（ecom-ref-gen）特有
- **读取前置清单为最高优先级**：产品信息/卖点/素材描述视为事实基线，100% 继承、不擅改任何产品参数/卖点文字/包装设计，生成范围严格限定清单内。
- 风格优先匹配电商场景（写实商业摄影/3D 渲染/清新电商风）。
- 产品三视图（正/侧/俯）；**所有文字、logo、包装参数与素材清单 100% 一致，文字清晰无乱码、不变形**。
- 产品变体写清变动在包装/规格/口味/腰封/丝带，且品牌文字 logo 型号不变。
- 带货主播/模特按角色处理（保持人脸体型一致，仅改服装）。

## 参考图 Prompt 质量增强（源自即梦 image-main，写提示词时套用）
- **抽象词必须同步给出具体画面翻译**，禁止单独使用：高级感→色彩克制(≤3色)+留白合理+材质统一+光线收敛；电影感→单侧主光+高光比+色彩分级+空间层次；氛围感→光线方向+色温+背景虚化程度+前景遮挡关系；有质感→具体材质的边缘高光+纹理层次+表面反射特征。
- **Prompt 写「如何实现」，不只写「要什么」**：❌「背景高级」→ ✅「背景采用低饱和深灰蓝渐变，中部干净衬托主体」；❌「很有质感」→ ✅「金属表面有清晰受控的高光条带，玻璃边缘有透亮折射，织物保留细密纹理与自然褶皱」。
- **主动规避廉价感**：主体不突出/背景脏乱无空间逻辑/光线无主光方向/高饱和乱堆/材质塑料感皮肤蜡感/五官手部结构透视不稳/特效无意义堆砌。
- **古风写实项目（题材限定，详见 `guofeng-makeup.md` 第七节）**：人物身份板/变体 prompt 补「写实质感后缀·身份板/单体档」（真实皮肤纹理、不过度磨皮、发丝清晰、织物刺绣细节丰富——纯白背景**不套**影调后缀），可在末尾追加古风负面词（`负面：二次元、AI假脸、塑料皮肤、现代网红妆、明星脸、假发感、头套感…`，防现代妆/假人感）；带环境的风格图/剧照类用「带环境档」完整影调后缀。妆造描述直接继承 Phase4 清单，**本阶段不重设计妆造**。回贴图 QC 时同步检查：妆容是否现代化/网红化、男性角色是否被加了浓妆。

## 强制输出规范
- 所有参考素材比例与 Phase6 全局比例一致（角色身份板、场景多视角设计板用 16:9 例外；产品三视图主体+变体用 3:2 例外；最终产品镜头才用全局比例）；
- 按「角色 / 物品 / 产品 / 场景 / 变体」分组展示生成提示词；
- 输出「参考素材确认清单」+「参考素材顺序编号表」（供 Phase8 `@图片N` 对齐）给用户核对。

## 红线
- 禁漏生成任何主体/变体；禁主体与变体核心特征不一致；禁部分生成完就推进（必须全生成 + 全确认）；禁用户改素材时文生图从零重做；禁用变体套变体；禁场景设计板任一格带人物/文字水印。

## 退出条件
**每张回贴图都有 QC 状态**（PASS / REGENERATED_PASS / ACCEPTED_WITH_RISK，无 FAIL_BLOCKED、无"非 PASS 且未记录风险"），质检报告已落盘，用户确认所有参考素材满意、无遗漏、无修改，且「参考素材顺序」已锁定 → 进 Phase8。

## 附录：原生 dreamina_cli 机制（接 CLI 自动生成时才用，手动模式可忽略）
- 工具：文生图 `text2image`、图生图 `image2image`，统一 `dreamina_cli` 提交；只要有可复用图输入（上传图/风格图/主体图/上一步结果）再次修改都必须 `image2image`，只有"从零首次生成、无任何可复用输入"才用 `text2image`。
- 模型版本：风格图 text2image `4.6` / image2image `4.5`；角色 image2image `4.6`；物品/场景 image2image `4.5`；**变体统一 image2image `5.0`**。
- 变体输入绑定：必须把步骤三主体图的 `output_id` 作为 `image2image` 主参考输入；参考主体则把真实 `resource_id` 传入 `image2image_param.resource` / `multi_modal2video_param.ref_images`；风格图只能辅助、不能是唯一输入。
- `dreamina_cli` 语义是"提交任务"，提交成功直接告知"已提交生成"，不主动轮询；完成标准是"所有应生成素材实际生成成功可被引用"，非"提交成功"。

# 横切规则速查（参考主体硬规则 + 回退表 + 引用写法 + 规则定值 + 命名）

> 本文件是 Phase1–9 通用的横切规则。任何阶段拿不准引用写法、回退到哪、取值多少，都查这里。

---

## A · 参考主体 vs 普通参考图（最易错的硬规则）

| 资产类型 | 唯一判定标准 | 后续处理 |
|---|---|---|
| **用户参考主体** | 被 `<subject>主体名</subject>` 标签包裹（或出现在 `<subject_list>` 块、早期等价写法 `@主体名`） | 直接复用为主体，**不重新生成主体图**，仅挖掘其变体；提示词里用 `<subject>主体名</subject>` 引用 |
| **用户普通参考图** | 仅在上传附件中出现、**没有 `<subject>` 标签**的图/视频/音频 | 仅作风格/外观参考，仍需后续生成标准主体素材，**不能直接当主体** |

> ❌ **绝对禁止反向猜测**：哪怕上传图明显是主角/主产品/主场景，只要没有 `<subject>` 标签，一律只能标「用户普通参考图」。这条贯穿 Phase4/7/8。

## B · 参考素材引用写法速查（Phase8 提示词里用）

| 资产类型 | 提示词里的写法 | 说明 |
|---|---|---|
| 用户参考主体 | `<subject>主体名</subject>` | 必须带标签，AI 才能锚定该主体；传真实 resource_id（手动跑即梦时即「在即梦里上传/选中的那个主体」） |
| 普通参考素材（风格图/场景图/道具图等） | `@图片N` / `@视频N` / `@音频N` | N 从 1 起，与「参考素材顺序」清单严格一一对应；手动跑即梦时，用户按该顺序上传素材 |

> **引用只有两种写法，由「有没有 `<subject>` 标签」唯一决定——这与 Phase4 的三态「主体来源」不是一回事**：
> - **来源三态**（用户参考主体 / 用户普通参考图 / 待生成主体）管的是「要不要生成、怎么生成」；
> - **引用两态**（`<subject>` / `@图片N`）管的是「提示词里怎么写」，会把三态收敛成两种：
>   - 有 `<subject>` 标签 / 在即梦登记为主体 → `<subject>名</subject>` + 传 resource_id；
>   - 其余全部 → `@图片N`，**包括 ① 用户普通参考图、② 待生成主体在 Phase7 出好的「标准主体图」**（出图后即按普通参考图编号引用，与官方 8-A 示例一致：司仪/顾承泽/沈念安等具名角色都写 `@图片N`）。
> - 可选升级：用户若在即梦把某待生成主体正式登记成主体（拿到 subject resource_id），可改用 `<subject>名</subject>` 获得更强身份锚定——**默认不这么做**。

**铁律**：
- 一个 Clip 中出现的所有角色/物品/场景，必须 **100% 显式引用**其对应参考素材，禁止纯文字提及。
- 引用**直接嵌入名词后**成为叙事一部分，不单独列在开头：
  - ❌ `参考@图片1的男主，滑到@图片2的展柜旁`
  - ✅ `男主@图片1以流畅滑行动作滑到卢浮宫展厅@图片2的展柜旁`
- **禁止代称**：已命名角色一律写全名，禁「他/她/女主/男主/两人/一行人」；多人同框逐个写清。
- 因为本机手动跑，提示词集顶部要附一份「**参考素材顺序**」编号表（@图片1=…、@图片2=…），让用户在即梦按同序上传，保证 N 不错位。

## C · 最小影响回退表（用户提改时按此定位，不全量重跑）

| 问题类型 | 回退到 |
|---|---|
| 剪辑/配乐问题 | Phase9 重新剪辑 |
| 单个 Clip 内容问题 | Phase8 重生成对应 Clip |
| **眼神方向错误 / 表情僵硬（成片后）** | **先走判责树再定层**（详见 `micro-expression.md` 视线锚点系统）：Prompt 已写明视线目标 → 重抽该 Clip 即可（种子/模型问题）；Prompt 没写目标 → Phase8 修 Prompt；Clip 表就没有目标/Transition → Phase5 补进 Clip 表。**不要一律按"单 Clip 内容问题→Phase8"处理** |
| **声音不对（开口假/语气平/千人一声/语速怪，成片后）** | **先走判责树再定层**（详见 `voice-control.md` 第八节）：Prompt 音色已含基础音色+声音过程 → 重抽该 Clip 即可；音色只写身份/裸情绪词 → Phase8 改音色描述；素材清单基础音色登记错误或多角色无差异 → Phase4 改登记 → 传导更新 Phase8 相关台词；台词标点节奏本身不对 → 回 Phase3/P5 改台词（标点变了须重过 5.2 计时） |
| **动作不自然（僵硬/滑行/站桩/纸片人/动作乱/多人只会对视，成片后）** | **先走判责树再定层**（详见 `action-naturalness.md` 第九节）：Prompt 已写清步态/身体联动/动作链 → 重抽该 Clip 即可；只写了"做什么"没写"怎么做" → Phase8 补步态+联动+守卫句；Clip 表本身没安排主动作/互动、或动作堆太多 → Phase5 按"1 主动作+1 辅助"配比调整分镜 |
| 角色/场景穿帮、素材不符合要求 | Phase7 修改对应素材 → 再重生成相关 Clip（必要时回 Phase4 补登记） |
| 分镜逻辑/Clip 切分/覆盖缺口 | Phase5 调整分镜 → 再重新推进后续 |
| 剧本/创意问题 | 回退到对应上游阶段（P3/P2/P1）调整 → 再重新推进后续 |

## D · 规则版本定值（附录 C，落地统一取这一套，不再纠结来源差异）

> ⚙️ 本节已**对齐即梦原生 SKILL.md 内核**（权威最高）；凡与早期主手册数值不同的，以下列为准。

1. **分镜要素**：取**五要素** = 景别 / 画面内容 / 镜头运动 / **镜头视角（拍摄角度，对应视频 prompt 的"从哪拍摄"）** / 台词旁白音效。景别用 **7 级体系**、运镜 10 种、角度 5 种，详见 `phase5-storyboard.md`。
2. **景别基础时长参考（无台词无动作）**：特写 **1~3s**、近景 **2~4s**、中景 **2~4s**、全景（环境/大场面）**2~6s**。（官方 shots-timing 参考值，仅参考、结合节奏与台词调整。）
3. **台词语速**：**每秒 5 字（含标点符号）**，说话时长 = 字数 ÷ 5；一边说话一边动作时，**计算时长 = max(说话时长, 动作时长)**（取较大值、不相加）；强制「台词必须自然说完不被截断」。
4. **Clip 切分阈值**：**装满优先**，尽量让每个 Clip 接近 10s 上限；累加达到 **8~10s 即封箱**（优先选剧情自然停顿处）；硬下限 4s、硬上限 10s；单分镜 <4s 必须与相邻分镜合并。
5. **Phase8 Prompt 结构**：用**官方 8-A 黄金结构**（`{整体风格}。生成一个由以下N个分镜组成的视频：分镜1：0-Xs，…{全局要求}{语言规则}`）。8-B「六段式」只是另一来源的等价归纳，落地输出用 8-A。
6. **时长策略**：**全片总时长 ≤10s → 只提交 1 次任务（整段生成）**；>10s 才拆 Clip 一次性并行（禁串行/禁分批）。
7. **参考主体引用**：素材清单/`ref-extract` 一律存 `<subject>主体名</subject>`；普通参考 `@图片N/@视频N/@音频N`，N 从 1、与素材顺序严格对应。注：原生 `video-prompt` 在视频 prompt 正文里把主体写作 `@主体名称`（与 `<subject>` 等价、都禁止退成 `@图片N`），落地以 `<subject>` 为规范、两种都识别。
8. **结尾约束**：以官方「全局要求段」为必备（禁字幕/BGM、配音留尾音、语言：中文）；早期「负向词段」（禁画面崩坏/五官变形…）为**可选增强**，可加在末尾但非官方必备。
9. **镜头字段写法**：官方五段式 `镜头：景别，画面落点，拍摄距离，从哪拍摄，运镜方式。`（如「全景，井道整体，远距离，高机位俯拍，固定镜头」）。

## E · 单镜与 Clip 时长硬约束（贯穿 Phase5/8，已对齐原生）

- 单个 Shot：**整数秒**（所有对外输出计时必须为整数秒，计算出现小数统一**向上取整**），上限 **10s**（>10s 超质量稳定上限，必须拆成子镜头）；计算时长 = max(说话时长, 动作时长)，说话时长 = 字数(含标点)÷5。
- 单个 Clip：**4~10 秒**（<4s 动作没完成；>10s 易崩坏），**装满优先：尽量接近 10s，累加 8~10s 即封箱**。
- ⚠️ **时长上限 10s 为用户实测覆写**：即梦官方原文是 15s，用户实测超 10s 质量崩坏（2026-06-11 收紧）。未来对照官方文档时**保留 10s，不要改回 15s**。唯一放宽通道：用户**明确要求冒险**时可对该次任务放宽到 ≤15s（平台上限），须明示质量风险并经确认，校验器传 `--max-duration 15`；默认值与常规装箱规则不变。
- 单个 Clip 内允许含 **1~5 个子分镜**（尽量 ≤5，避免 AI 信息过载）。
- Clip 总时长 = 所含 Shot 计算时长之和（整数）；全片总时长与预估误差 ≤ ±10%。
- 比例全局统一（Phase6 定值），所有素材/片段/成片同比例；**角色身份板、场景多视角设计板用 16:9 横屏；产品三视图（主体+变体）固定 3:2 横屏**（均为参考资产，板式需要、不随 Phase6 全局比例）；**最终产品镜头（Phase8 视频）才采用全局比例**。

## F · 命名规则

- 项目名仅保留中文 / 英文 / 数字，其他字符替换为下划线。
- **归档**：所有产物写进固定根目录下的项目文件夹 `C:\Users\Administrator\Desktop\claude长视频创作\{项目名}\`（详见 SKILL.md §3），不散落到桌面/当前目录。
- 产物文件名（均置于该项目文件夹内）：`{项目名}_剧本.md`、`{项目名}_素材清单.md`、`{项目名}_故事板.md`、`{项目名}_视频生成Prompt集.md`；成片名 `最终成片_{项目名}_{时间戳}`。

# Phase 4：素材挖掘（ref-extract）— 原子素材识别 + 素材深化设计

> 本阶段是**「素材识别」+「素材深化设计」二合一**：既挖出要视觉化的元素，又把每个主体/变体写到「足以直接支撑后续参考素材生成（角色身份板 / 产品三视图 / 场景多视角设计板）」的完整视觉设计。产物 `{项目名}_素材清单.md` 同时承担"素材识别结果"和"参考素材清单"两种职责。
> 边界：本阶段**不出参考图 prompt、不做跨 Clip 隐式道具补件**（那是 Phase7 / Phase5.4 的活）。

## 前置触发（满足任一启动）
- ✅ Phase3 完整剧本已得用户明确确认；或用户直接给了可用的完整剧本 / 分镜列表 / Clip 表。
> 硬规则：上游产物未经用户确认前，禁止启动。

## 核心原则
1. **输出全量重要素材，不做流程假设**：只根据当前输入挖，不因"后面可能补"而省略已明确出现的重要素材。
2. **主体必须 = 变体列表第一项**（角色/物品/产品/场景都适用，排序硬规则）。
3. **用户参考主体优先复用**：只有被 `<subject>主体名称</subject>` 包裹 / 在 `subject_list` 里的才是参考主体，引用保留原 `<subject>` 格式、禁止改普通名称；其余上传图（`<upload>`）一律「用户普通参考图」，**禁止反向猜测**（错误推理：「上传了参考图且是主角，所以是参考主体」）。
4. **每条必须显式标 `主体来源`**（用户参考主体 / 用户普通参考图 / 待生成主体），禁止留给下游猜。
5. **纯风格参考图不自动变素材参考（与 Phase1 风格卡联动）**：Phase1 风格卡 `是否兼作场景参考＝否` 的上传图是**纯画风参考**，**禁止登记为任何角色/场景/产品的「用户普通参考图」**；只有该字段写 `是，指定为场景X / 角色X / 产品X` 时，才**仅**把它登记为**指定的那一类那一个**，不得被其他类吃掉（防"本来只当开场场景、却被角色参考串线"）。AI 不替用户猜「是」，默认按「否」处理。

## 一、四大类挖掘规则

### 1. 角色挖掘
**必挖**：主角、配角、所有有台词角色、推动剧情/卖点的功能角色、多次出现需稳定形象的角色、营销里承担展示/讲解/试用/对比/见证的人物。
**不挖（群体背景角色）**：背景宾客、围观群众、路人群、同学群、台下观众等纯环境填充角色。
- `小队队员/士兵们/随行下属/宾客们/黑衣人一行` 这类**群体称呼，禁止作为一个条目写进清单，更禁止标 `待生成主体`**。
- 若群体里有明确分工或镜头给到清晰辨识 → 拆成单体（队员A/队员B、保安1/保安2、面试官1/面试官2）；做不到或无需拆 → 不进清单，保留在剧本/分镜的群像描述里。

### 2. 物品/道具挖掘（从严，宁缺毋滥）
**只挖值得单独做稳定参考主体的物品。**必挖：关键剧情道具、被镜头强调的物品、前后反复出现需保持一致的物品、营销里演示功能/形成对比/体现卖点的物品。
**默认不挖**：普通环境杂物（杯子/纸张/书堆/摆件/文具/装饰画/窗帘/餐具/花瓶）、一次性随手物件、纯生活气息填充物、纯背景陈设。
- **一次性动作道具不挖**：临时凶器、顺手抄起的杂物、短暂递出的生活物件（如杀人用的银针、临时刺人的发簪、一次性泼出的药碗），后续不再持续出现/不作证据物保留就不挖。
- **口头提及但当前未出现的未来对象不挖**（「待会儿去吃肉包子」「明天去买花」当前没实际出现 → 不挖）。
- **复核铁律**：出清单前逐个自查"这个物品是否真的对剧情有重要作用"，答案不是明确「是」就删掉。

### 3. 产品挖掘（营销类强制，独立成一级分类）
只要是营销/带货/广告/品牌传播/产品演示内容，**必须单列产品清单**。必挖：主推产品、套装核心产品、关键 SKU/型号/口味/包装规格、脚本明确重要的赠品/礼盒/替换装、当前已实际出现/展示/试用/对比的产品。
- 以卖点传播/品牌识别/购买转化为核心的 → 归**产品**；仅作使用环境辅助道具 → 归**物品**。

### 4. 场景挖掘（只挖重要故事场景）
必挖：发生关键剧情/卖点表达的场景、角色首次出现/关系变化/动作转折/产品展示所在场景、营销的使用/痛点/对比/品牌场景。过渡通道、背景陪衬、短暂掠过的地点**不挖**。
- **场景拆分变体**：同一大场景内若空间功能/视觉结构/天气（明显且稳定）/机位逻辑/用途不同 → 拆成场景变体（客厅-沙发区 / 客厅-餐桌区；晴天庭院 / 暴雨庭院 / 大雪庭院；白天营业 / 深夜打烊）。
- **🟡 场景人口基线（必标，治"空场景"病根）**：场景**参考图永远空镜无人物**（空间设计板，见三-场景设计），但「成片里这个空间该有多少人」是**另一回事**，必须在素材清单的每条场景里显式标一个 `场景人口` 字段，**否则下游 Phase8 会照着空场景图生成无人画面**（教室没学生、餐厅没食客、街道没行人——最高频穿帮）。判定看**该空间在该剧情时段的常识满载度**：
  - `空镜无人`——剧情明确无人（深夜空教室、打烊店铺、清晨无人街、私人书房独处）；
  - `仅主要角色`——封闭/私密场景只有剧情角色（会议室谈判、卧室对话）；
  - `主要角色+背景群像`——公共/营业场景按常识本就有人（**上课的教室有学生、营业餐厅有食客、白天站台/街道有行人、宴会有宾客**），标明群像身份+密度（稀疏/半满/满座/拥挤）。
  - 背景群像**不登记为角色主体、不出参考图、不给台词**（它是环境，不是角色——与本节「群体背景角色不挖」一致）；同一场景不同时段人口不同就**随场景变体分别标**（白天满座教室 / 放学后空教室）。此字段是 Phase8「分镜人口声明」与自检闸 8 的上游依据。

## 二、真变体 vs 伪变体（最易错，务必判准）

**变体只保留差异巨大、稳定、可复用的版本。**

| | ✅ 真变体（要挖） | ❌ 伪变体（不挖） |
|---|---|---|
| 角色 | 童年版、老年版、严重受伤/重度烧伤版、明显怀孕版、截然不同的成套服装版、伪装身份版、非人化版 | 姿势/动作过程、持物、临时受限、轻微情绪、短时表情、轻微脏污/淋湿、普通哭笑、戴摘普通配饰（"手持血书版""持剑对峙版""坐下版""回头看版"都不算） |
| 物品 | 开/关、完整/破损、空/满、点亮/熄灭、包装前/包装后、使用前/使用后 | 在手中/桌上/掉地上/被举起/供桌摆放/左手拿（位置与持握状态不是变体） |
| 场景 | 白天/夜晚、营业中/打烊后、整洁/凌乱、晴天/雨天（明显稳定的空间/天气变化） | 天空微暗、云层略厚、普通亮度波动、花瓣飘过等模型可自生成的细小变化 |

- **服装归类边界**：人物穿在身上的校服/婚衣/嫁衣/外套/礼服/盔甲，**默认属角色主体或角色变体，不入"物品"**；只有当服装脱离身体、作为可摆放/可拿取的独立对象存在时，才归物品。
- 变体描述必须**把变化点写具体**：哪里变、变成什么、哪些不变。不能只写"治愈版/受伤版/雨天版/节日版"这类空标签。

## 三、素材深化设计（每条都要写到能直接生成对应参考素材：角色身份板/产品三视图/场景设计板）

### 角色设计
- 默认**亚洲面孔**（除非背景明确要求）。
- 主体与每个变体描述都必须**从头到脚、含足部**（鞋/赤脚），足以直接支撑角色身份板生成。
- 明确 **4~6 个关键识别特征**；不同角色**主动拉开差异**（性别/脸型五官/发型/体型/穿搭气质多维拉开，避免"只换衣服的同脸角色"）；必写性别 + 面部（脸型、眉眼、鼻/唇多维）。
- 若来自用户普通参考图：可参考该图提炼外观，但写入时只写**提炼后的角色描述**，禁止写"参考@图片1的女生外貌"这类句子。
- **古风写实项目（题材限定钩子，详见 `guofeng-makeup.md`）**：角色妆造按性别分流——男性默认**面部干净无浓妆**，重点写发型/服装/配饰（高马尾=利落、发冠=有家世、盔甲=将军、长袍=公子）；女性按**八大维度**写清 底妆质地/眉形/眼妆/腮红/唇色/花钿额饰/妆面完整度/皮肤质感，强度对齐身份层级（民女近素颜→闺秀日常精致→宫宴礼仪完妆），32 个角色方向库可直接取用；妆容颜色与全局影调统一（深宫暖色配红唇金饰、冷青清冷配浅唇银饰）；**状态残妆按其第五节"真伪变体分级"判定**——战损版/失势版等跨场景稳定状态才登记变体，刚哭过/雨水晕妆等瞬时状态走 Phase5.4 状态延续、不挖变体。现代戏/风格化画风项目不套用。
- **有声主体补音色**（见下）。

### 物品设计
- 分类标 `静态陈设(Static Prop)`（属场景环境、后续整合进场景）/ `动态道具(Dynamic Prop)`（角色携带/拾取/使用）。
- 每条描述完整自包含，严禁"同上""与XX一样"；变体写清具体变化部位与保持不变部分。

### 产品设计（营销强制）
- 出**产品文字信息清单**：上传图则完整提取品牌名/型号/logo文字/参数/卖点；未上传则提取用户文字里的产品核心信息。
- 主体/变体都要支撑三视图；变体写清变动落在包装/规格/口味/外盒/瓶身哪里 + 哪些品牌文字/logo/型号不变。

### 场景设计
- **空镜、无人物**（后续 Phase7 做成空场景多视角设计板）；整合该场景所有静态陈设并嵌入场景描述；写清剧情功能、时间段、天气/光线、空间结构、关键陈设、可见痕迹、氛围；变体写清变化维度与落点。
- **⚠️ 区分"参考图空镜"与"成片人口"**：这里写的"空镜无人"只约束**参考图**（空间设计板必须无人，否则每镜人物乱、背景跑脸）；**成片该有多少人是另一回事**，由上面「场景人口基线」单独标注、由 Phase8 逐分镜落地。**别因为参考图空镜，就默认成片也该是空场景**——这正是"上课教室生成空教室"的病根。

### 音色描述（有声主体必写，固定格式）
- 格式：`性别__，年龄__，语调__，音高__，语速__，口音无`，**年龄必须精确整数**。
- **扩展字段（详见 `voice-control.md` 第一节）**：官方 6 字段后可追加 `声线质感__，情绪底色__，发声习惯__` 三项（如 `声线质感清冷微哑，情绪底色克制隐忍，发声习惯：咬字清楚、句尾轻收`）——**有台词的主要角色建议补全**（Phase8 的声音结构靠它做基础音色），龙套/功能角色官方 6 字段即可；扩展字段写**可听参数**（干哑/清亮/停顿/咬字/尾音/气息），不写比喻、不写"声音像{名人}"。
- 同一视频多个声音主体须**显著差异化**（≥4 个字段明显不同），优先拉开年龄层/语调气质/音高/语速；扩展字段同样参与拉差异。
- 挂在对应角色条目下；独立旁白可单设 `角色：旁白` / `角色：品牌旁白` 承接音色。

## 四、VFX 视觉机制 / 状态规则（可选扩展，非原生四类）

> 这一类**不在即梦原生 ref-extract** 里，是《浅溪石阶》案例的补充。当故事依赖「跨镜头必须统一、但不是实体角色/道具」的视觉语义时启用；普通写实题材可不写。
- **VFX 视觉机制**：自定义视觉符号 + 颜色/形态语义（冷灰蓝半透明残影＝过去脚步；暖橙红虚线闪烁＝未来影；静水镜面 vs 浑水边界）。
- **状态规则**：跨镜头延续的状态变化（右肩破损渗血、墨镜、血迹），与 Phase5.4「人物状态持续标注」打通。
- 启用时建议作为第五个一级分组 `## VFX视觉机制 / 状态规则`，每条写 `规则说明`（颜色·形态·语义 + 跨镜约束）。

## 五、输出格式（统一 Markdown 层级）

- 文件标题 `# {项目名}素材清单`；一级分类 `## 角色 / 物品 / 产品 / 场景`（营销才有产品；可选 `## VFX视觉机制 / 状态规则`）；每条 `### 资产名`；字段用加粗标签。
- `---` 分隔线前后各留一空行。
- **必须落盘**为项目文件夹下的 `F:\claude长视频创作\{项目名}\{项目名}_素材清单.md`（归档规则见 SKILL.md §3）并展示给用户确认；不允许只在对话里给、不落盘。

**字段模板**：
```markdown
# {项目名}素材清单

## 角色
### 角色名
**类型**：主角 / 配角 / 功能角色
**性别**：男 / 女
**主体来源**：用户参考主体 / 用户普通参考图 / 待生成主体
（若为用户参考主体，补）**参考主体**：<subject>主体名称</subject>
**主体**：默认形象名（=变体列表第一个）：完整角色描述（从头到脚含足部）
**变体**：变体B：具体改动与不变点；变体C：…（无变体则不写本行）
**音色描述**：性别女，年龄24，语调轻柔克制，音高偏低，语速偏慢，口音无，声线质感清冷微哑，情绪底色克制隐忍，发声习惯：咬字清楚、句尾轻收（无声主体则不写；后 3 项为扩展字段，主要有台词角色建议补全）

## 物品
### 物品名
**物品类别**：道具 / 陈设 / 使用物 / 证据物 / 包装物
**主体来源**：…（参考主体补 **参考主体**：<subject>名</subject>）
**主体**：默认版本名（=变体首项）：完整物品描述
**变体**：…（无则不写）
**分类**：静态陈设 / 动态道具

## 产品
### 产品名
**产品类别**：护肤品 / 饮料 / 家电 / 食品 / 软件服务
**主体来源**：…
**产品文字信息清单**：品牌名、产品名、容量、卖点文字…
**主体**：默认版本名（=变体首项）：完整产品描述
**变体**：…（无则不写）

## 场景
### 场景名
**场景类型**：主场景 / 子场景 / 功能场景
**主体来源**：…
**场景人口**：空镜无人 / 仅主要角色 / 主要角色+背景群像（注明群像身份+密度，如"满座低头自习的学生"）——参考图仍空镜，本字段约束成片人数
**主体**：默认场景版本（=变体首项）：完整场景描述（嵌入静态陈设）
**变体**：…（无则不写；同场景不同时段人口不同时随变体分别标场景人口）
```

## 输出约束（红线）
- 覆盖所有重要素材，不漏配角/关键道具/主产品/关键场景；物品/场景只列重要的，不堆背景杂物与过渡空间。
- 群体角色不得作为一个 `待生成主体` 条目；服装不当物品；位置/持握/动作不当变体。
- 每条显式 `主体来源`；参考主体保留 `<subject>` 原格式。
- 角色必写 `性别` + 具体面部 + 拉开差异；场景描述详细到可直接生空场景多视角设计板；营销必单列产品。
- 错误写法：`女主：上传了参考图、是主角，所以主体来源是用户参考主体` ❌
- 正确写法：`女主：上传了普通参考图，对应主角外观；未见 <subject> 标记，故主体来源是用户普通参考图` ✅
- 本阶段不出参考图 prompt、不做隐式道具补件。

## 退出条件
用户确认素材清单无遗漏、无错误、主体来源标注正确 → 进 Phase5。

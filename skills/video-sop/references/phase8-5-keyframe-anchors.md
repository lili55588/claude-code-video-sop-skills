# Phase 8.5：即梦片段首帧/关键帧宫格锚点包（keyframe-anchors）

> **位置**：Phase8 视频生成 Prompt 已**校验通过并确认** → Phase8.5 片段视觉锚点包 → 用户拿去即梦生成视频 → Phase9 剪辑。生成结果校验仍在 Phase8「八、生成结果校验」里，本阶段不接管。
> **定位**：为每个 Clip 出一张可直接上传即梦的 `5/6 宫格编号分镜图`，给即梦一个紧凑、有序的**构图/调度参考**，降低抽卡。**它只锁"画面顺序、站位、动作节奏、转场"——身份/场景细节/画质仍由 Phase7 参考图负责**。
> **性质**：这是本机工作流的工程增强，**不是即梦官方原生阶段**，别当平台规则讲给用户。
> **可选阶段**：项目不需要锚点包时可整段跳过，直接拿 Phase8 Prompt 裸跑即梦。

## 🔧 本机执行模式（与 Phase7/8 一致：本机不出图）
本机没有 `text2image / image2image`，所以**关键帧图不在这里生成**。本阶段产出：① 每个 Clip 的「锚点计划 + 逐帧图生图提示词」；② QC 清单；③ Phase8 Prompt 增补。实际出关键帧图走「**用户即梦手动跑** 或 **交 Codex 出图**」（双引擎分工：Claude 主笔提示词/计划/拼接/QC，Codex 出图）。用户回贴关键帧 PNG 后，**本机做两件不算"生图"的确定性活**：逐图 QC（可直接看图）+ **本机像素级拼宫格**（PIL / ffmpeg，ffmpeg 可复用 video-auto-edit 工具箱）。

> **首帧/新机位关键帧走标记式两阶段（`marked-camera-anchor-workflow.md`）**：本机用 `scripts/mark_image_annotations.py` 画相机/角色标记（离线、不生图，算第三件确定性活）；出机位底图/角色落位图仍是**生图**——内置图片工具可用就直接调，不可用则产"标记图 + 无标记原图 + 提示词"标 `WAITING_EXTERNAL_IMAGE` 等用户/Codex 出图回贴。**默认内置生图、不静默切外部生图脚本/小云雀、标记软引导优先于 Mask。**

## 二、张数预算
- **推荐 5 张/Clip，硬上限 6 张/Clip。**
- 超过 6 张才说得清 → **回退 Phase5/Phase8 拆分或简化 Clip**，不硬塞进宫格。
- **默认不为每个子镜头都出图**。只出能控连续性的锚点：开场状态、主要动作切换、镜头/空间切换、情绪/动作转折、结尾状态。
- 一个子镜头可出 0 / 1 / 2 张，**最多 2 张**；需要 2 张时用「动作中段 + 结束状态」，**不能变成两个不相干的机位**。
- 「何时给一镜 2 张」沿用分镜判据：**景别与运镜都不变→1 张；景别或运镜变了但动作不激烈→2 张；动作激烈→本就该在 Phase5 独立成镜**。

## 三、帧角色
- **Frame01 永远是该 Clip 的真·首帧图**，严格匹配 Phase8 该 Clip 的**开场状态**：角色站位、画面人口、景别构图、视线、手、道具、光线、环境。首帧没有上一帧，用**角色图+场景图+道具图多图合成的图生图**（不是文生图）。
- **Frame02+ 是关键帧图**，按真实视觉变化决定：动作阶段、站位变化、镜头变化、情绪转折、道具状态变化、跨 Clip 衔接。
- **每个 Clip 的最后一张通常代表结尾状态**，尤其当下一 Clip 需要严格承接时。
- **跨 Clip（Clip 2+）**：新 Clip 首帧要么**严格承接上一 Clip 尾帧**（需要连贯时），要么按 Clip 表的自然硬切用更松的「同场景、新动作 beat」关系。**不得无可见转场就跳变**位置/左右手/视线方向/光向/道具位置/场景地理（KB 首尾帧语法：下一单元起点=上一单元终点不跳步）。

## 四、参考优先级铁律（防漂移累积，最重要）

> 🔧 本阶段验收（闸9）挂**视觉验收内核 VVK**（见 `visual-verification-kernel.md`）：身份锚 P7 角色图（非上一帧）、站位锚场景图；**只能引用 `LOCKED_PASS`/带风险的 P7 素材**（`REJECTED`/未锁定图连带阻断）；宫格 artifact/编号防泄进成片走软信号检测 + 返修（写区域坐标）。

逐帧用图生图，**但严禁让已生成的帧成为身份来源**。优先级：

1. **身份权威** = Phase7 角色图 / 已确认状态变体。脸、体型、发型、**服装设计**、永久配饰、角色专属道具，**只取这里**。
2. **场景权威** = Phase7 场景图 / 已确认环境板。空间布局、地标、材质、光向、尺度，取这里。
3. **道具/产品权威** = Phase7 道具/产品图 / 已确认变体。形状、标记、颜色、可读文字、材质，取这里。
4. **连贯参考** = 上一帧 / 上一 Clip 尾帧。**只能贡献**：位置、身体朝向、视线、动作动势、道具位置、光向、地面痕迹、灰尘/水渍/脚印、临时状态。

> 🔴 **绝不**用已生成的关键帧当角色的脸/服装设计/身材比例/道具设计来源——那就是漂移累积（storyboard-rag 实测放弃逐帧串的根因）。

**衣服必须拆**：
- 衣服**设计/图案/版型/颜色身份** → 取 Phase7 角色图 / 服装状态变体。
- 衣服**状态**可跨帧继承：湿/破/脏/皱/血污/落灰/松散/被风吹。

Clip 用到 Phase4/Phase7 的状态变体（受伤/伪装/礼服/产品开盖/道具破损/爆炸后损伤）→ **以该变体为身份权威**，不取默认主体图。

## 五、跨帧空间锁：道具位置图 + 角色站位图（防无过渡漂移，根治闸）
> 病根（真实翻车，2026-06-17「重返那年盛夏」Clip1）：即梦/生图会让**稳定道具的承载面/容器、角色的物理站位**在相邻关键帧间"无动作漂移"——道具不经搬移就从桌面跳进纸箱、角色不经走动就从书桌正面变到侧面。这不是单条 Prompt 的问题，**出帧前必须先锁两张 Clip 级 Map**。

### 5.1 道具位置图（Prop Position Map）
出任何帧前，为**跨帧出现**的稳定道具/容器/文件/武器/产品/手持物/重要家具各写一行，字段至少：
- 道具名 + 权威参考图；
- 初始承载面或容器：桌面 / 地面 / 纸箱 / 行李箱 / 角色手 / 口袋 / 架子 / 托盘 / 桌角…；
- 稳定相对位置：里面 / 上面 / 旁边 / 左右前后 / 靠窗侧 / 纸箱边缘…；
- 接触点与归属：躺在盒内 / 右手持握 / 指尖捏住 / 压在纸上 / 靠着箱壁…；
- 允许移动：保持不动 / 被某只手拿起 / 放到某表面 / 移入纸箱 / 打开合上 / 左右手交接。

🔴 **硬规则**：道具**不得无可见动作就改变**承载面/容器/左右位置/归属手/接触点。Frame01 铅笔盒在桌上、Frame04 在纸箱里 → 中间**必须有一帧可见"从桌面移入纸箱"的动作**，否则锚点包判 `FAIL_BLOCKED`。
**容器逻辑要拆开**：「铅笔在桌面铅笔盒内」「铅笔从盒中被拿起」「铅笔盒在纸箱内」是**三种不同空间状态**，无可见过渡不得混为一谈。

### 5.2 角色站位图（Character Blocking Map）
只要命名角色**跨帧出现**、或其站位对动作有意义，出帧前各写一行，字段至少：
- 角色名 + 权威参考图；
- 起始物理位置（相对场景地标）：书桌正前方 / 纸箱旁 / 门左侧 / 靠窗侧 / 某角色身后 / 第二排课桌…；
- 身体朝向与面对目标：面向书桌 / 面向窗 / 面向某角色 / 背对门 / 侧身对镜头 / 蹲向纸箱…；
- 画面左右关系与轴线：与其他角色/地标的真实左右关系、180° 轴线在哪侧、**是否只是摄影机移位而角色不动**；
- 视线与互动目标：看手中道具 / 看某角色 / 看窗外 / 低头看纸…；
- 允许移动：保持不动 / 前倾 / 转头 / 转身 / 坐下 / 起身 / 侧移 / 绕桌 / 进出画 / 从正面走到侧面 / 与另一角色交换左右。

🔴 **硬规则**：角色**不得无可见动作就改变**物理位置/地标所在侧/朝向/左右关系/坐站蹲状态/互动接触点。Frame01 在书桌正面、Frame02 在书桌侧面 → 要么写出"绕到书桌侧面"的可见动作，**要么明确"仅摄影机移到侧面拍摄，角色仍在同一物理位置"**，否则 `FAIL_BLOCKED`。
**机位变化 ≠ 角色移动**：换角度只是从另一侧拍同一固定位置，单帧提示词必须写明"角色相对桌/窗/门/箱仍在同一物理位置"；绝不让换角度悄悄变成角色挪位。多人镜头：真实左右/前后景/过肩/距离/谁面对谁，**在剧本给出可见的越轴/转身/靠近/后退/换座/进出之前一律保持**；无解释的左右互换或远近跳变=站位失败。

### 5.3 机位锚定图（Camera Anchor Map）—— 防机位悬浮 / 防"可见域"被当目标硬凑（根治闸）
> 病根（真实翻车，2026-06-18「重返那年盛夏」Clip4）：分镜写"陈默后排斜侧主观视角看苏晴侧后与侧脸"，生图把"侧脸"当**结果目标**去凑→苏晴转头/歪脖子/换手摆拍；相机飘在课桌列与列之间的过道、"和谁的桌椅都不对齐"=机位悬浮。**根因：相机没被钉成场景里一个可指认的物理对象，可见域没从机位几何推导、而是被当愿望填写。**

**触发条件（命中任一就必填本图；纯正前方/平直 establishing 镜头可只填一行轻量机位锚）**：分镜含 `主观视角 / 过肩 / 斜侧 / 特定机位角度 / 越 180°轴 / 可见域需从机位推导（如"侧后与侧脸""只露轮廓"）`。

出帧前为该镜写一行，字段至少：
- **相机身份**：它是谁的视角 / 在哪个物理对象上（某角色座位 / 某角色肩后 / 门口 / 固定机位…）；
- **物理锚点坐标**：钉在场景可指认地标的网格/相对坐标（如"内侧 c4 列·苏晴后方第 3~4 排座位"、"陈默右肩后 0.5m"）；**不得是空中悬浮点 / 过道无座位点**；
- **眼高 / 角度**：坐姿眼高平视 / 站立俯拍 / 低角度……；
- **视线向量**：从锚点指向被摄主体的方向（如沿网格对角线斜向前-右）；
- **前景锚**：能坐实"相机在矩阵里"的近景物（前排同学椅背/桌沿一角 / 肩膀 / 门框）；
- **可见域（涌现量，非愿望）**：由上面几何**推导**出主体能被看到多少（侧后轮廓+一缕脸颊边缘 / 全侧脸 / 仅背影）。

🔴 **硬规则**：
- 相机必须能在画面里被**反向指认**——"它坐在/站在场景哪个可指认位置"。指认不出（前景与网格不收敛、机位悬浮于过道/空中）= `FAIL_BLOCKED`。
- 可见域是机位**几何的结果**，不是目标。**禁止为凑可见域让被摄角色转头/歪脖子/扭身/换手摆拍**；出现摆拍 = `FAIL_BLOCKED`。
- 机位变化沿用 5.2「机位变化≠角色移动」：换角度=从另一物理点拍同一固定主体，角色不挪位。

**🔧 执行层·标记式两阶段证据链（升级，见 `marked-camera-anchor-workflow.md`）**：本图字段是**文字规格**；**首帧与引入新机位的关键帧，把它落成实际图像证据链**——① 在场景全景图上用标记脚本画相机框（红框=相机**眼位**物理坐标）+ 视线箭头 → 生成**干净机位底图**（无角色），通过标准 = 能从近景遮挡/地标视差/距离压缩/左右关系**反推相机确实在标记原点**（只从旧全景转个向、原点没动如"教室后方高处俯视全班" = `FAIL_BLOCKED`）；② 在机位底图上画角色框（绿框=真实支撑点）+ 身体朝向箭头 → 加角色身份/人口/道具。同机位后续关键帧**复用已验收底图、不重猜空间**；小问题走标记式局部返修（小红圈只修一处）。**默认内置生图、标记软引导优先 Mask、未经用户点名不调外部脚本/小云雀。**（相机高度写"坐姿/站姿眼平"，**别用"眼高"二字**——"高"会被生图模型读成高角度俯视。）

> 🔸 **「必须看见 X = 阻断」的适用边界（防误伤可读镜头）**：本条只禁**与所选机位几何矛盾、该机位本就拍不到**的点名（被遮的侧脸 / 被身体桌沿遮住的握笔手 / 角色背后的物体）。**镜头本身就为"让某物清晰可读"而设的——产品特写/广告、文字/logo、招牌、道具识别、剧情必读文字——清晰可读是合法且必需的生成目标（且与 VVK「OCR 二钥」一致），不在禁止之列。** 判据：要的可见性是**顺着机位**、还是在**跟机位对着干**。

## 六.5、生图前置考量 · 镜头设计 8 维（写提示词前逐帧走一遍）
> 链路：**§六 选哪些帧 → §6.5 逐帧把设计想全 → §七 填规格块 + 逐字段落笔**。本节是动笔前的设计检查，**喂** §5 两张 Map 与 §七规格块；§3「镜头语言」想清的焦段/构图/景深/光线再回写进 §七「镜头」字段。
> **轻重原则（对齐闸9，别把每帧写成八大段）**：简单平直镜头允许标 `NA / 沿用上一帧`；**带主观视角 / 斜侧 / 过肩 / 多人站位 / 道具接触 / 跨帧连续的镜头，必须声明相关维度**。

按决策依赖排序（前者约束后者）：

0. **继承 + 帧角色 / 动作时相**
   - 继承：Phase8 分镜已定的景别/机位/角度/运镜 + Phase7 身份/场景/道具权威 + 上一帧连贯锚——**先抄准、不自创**；
   - 帧角色：本帧是 **首帧 / 动作中段 / 转折帧 / 尾帧**？
   - 动作时相：动作处在 **起 / 中 / 止** 哪一刻？
   - 承接：承上一帧**什么**、交下一帧**什么**。
1. **空间 · 站位**：地标、180°轴、角色物理位置+朝向+视线锚、画面人口/背景层次、道具承载面。（§5.1+§5.2+闸8）
2. **机位**：谁的视角、物理锚点坐标、眼高、距离、视线向量、前景锚。（§5.3）
3. **镜头语言**：景别、角度、**画幅/裁切安全**、**主体画面占比**、**焦段感（广角扩张 / 长焦压缩·隔离主体）**、运镜、**构图重心（主体位置/头上空间/视线前留白/前景遮挡/对称失衡）**、**景深焦平面（深焦/浅焦、焦平面在谁、虚化层）**。
   - 🔴 **焦段变化 ≠ 机位移动**：要"后排主观视角 + 近景"，靠**长焦压缩 + 浅景深**把主体拉近隔离，**不是把相机挪到主体旁边**（挪相机=机位/站位变了，违 §5.3）。
4. **可见域 · 遮挡**：由 1+2+3 **几何推导**主体露多少、什么被挡；描述因不描述果。（§5.3+六原则）
5. **光线 · 调性**：光源方向（有据：窗/灯/日）、质感（硬/柔）、key/fill/rim、色温时段 + **项目色彩编码**（如本片倒叙：现在=冷低饱和 / 回忆=暖金高饱和）+ 与邻帧光向连续。
6. **表演 · 情绪**：微表情（脸/眼 → `micro-expression.md`）、身体动作（→ `action-naturalness.md`）、口型张合（→ `voice-control.md`）；**情绪只用可见身体线索表达**（净化）。
7. **连续性入口 / 出口**：
   - **入口**：锁上一帧的 位置 / 朝向 / 光向 / 道具状态；
   - **出口**：锁下一帧 / 下一 Clip 要承接的 位置 / 朝向 / 视线 / 光向 / 道具位置；
   - 有无**可见过渡**。（§4+§5）

## 八、宫格拼接（仅本机像素级拼接，禁 AI 重绘）
单帧过 QC 后，**只用本机确定性合成**拼最终宫格：
- 工具：PIL / ImageMagick / ffmpeg（ffmpeg 可复用 video-auto-edit）；
- ❌ **绝不**让 AI 生图模型「把这几张拼成宫格」——那会重画已通过的帧、作废 QC；
- 除按约定面板比例缩放/裁切外，**逐格保留已通过的帧原样**；
- 面板顺序**从左到右、从上到下**可读；
- 默认版式：5 张 = 上 3 下 2（或别的稳定 16:9 版式，保证脸够大）；6 张 = 3×2；
- 少用粗边框，必要时细分隔线即可。

### 编号策略
两种模式由用户选：
- **`编号宫格模式`**（**本项目默认可用**：用户已实测即梦不会把小号面板编号渲进成片）——编号要小、固定在角落或安全留白条，**绝不遮脸、手、道具接触点、画面文字、字幕区**。
- **`干净宫格模式`**（换模型/换项目未复测时的保险）——不画编号，靠从左到右从上到下顺序 + Prompt 措辞表达顺序。

> ⚠️ 无论哪种模式，**Phase8 Prompt 增补里都必须写「宫格仅供规划参考、不要把宫格本身/编号渲进成片」**。这跟「编号模式可用」不矛盾——编号是给人/上传看的，成片里仍不该出现它（防即梦版本/模型漂移）。除非用户明确要工业故事板，宫格里不放箭头/标签/标题/时间标注；若放了，必须在 Phase8 增补与负面词里排除其渲染。

## 十、锚点 QC 闸（对应闸 9，每 Clip 收尾给一个状态）
四态：`PASS` / `REGENERATED_PASS` / `ACCEPTED_WITH_RISK`（须记录具体风险 + 用户明确接受）/ `FAIL_BLOCKED`（**该 Clip 禁上传即梦**，重生失败帧/宫格或回退最小受影响 Phase）。

逐项查：
- **身份**：脸/发型/体型/服装设计/永久配饰随 Phase7 权威图；
- **状态变体**：受伤/伪装/礼服/道具状态用对了变体；
- **场景**：地标/出入口/路径/桌椅/光向/尺度/材质/背景人口与 Phase7/5/8 一致；
- **角色站位图（5.2）**：每个跨帧命名角色逐帧都声明了相对地标位置、身体朝向、画面左右关系、视线目标、允许移动；**无解释的"正面→侧面/近→远/左→右/坐→站→蹲/前景→后景/面向窗→面向门"跳变 = 阻断**（除非写了可见动作或"仅摄影机换位"）；
- **道具位置图（5.1）**：每个跨帧道具逐帧都声明了承载面/容器与接触点，任何变化都有可见动作帧解释；**无解释的"桌面→纸箱/手→桌面/左手→右手/容器内→容器外/打开→合上/前景→后景"跳变 = 阻断**；
- **机位锚定图（5.3）**：带机位分镜逐帧声明了相机物理锚点/眼高/视线/前景锚/可见域(涌现)；**相机在画面里指认不出物理位置（悬浮于过道/空中、前景与网格不收敛）= 阻断**；**为凑可见域出现转头/歪脖/换手摆拍 = 阻断**；**提示词正文出现"必须看见 X"等与机位矛盾的结果目标 = 阻断（应改写成"因"；为可读而设的产品/文字/logo/道具识别镜头除外，见 5.3 🔸）**；
- **标记式两阶段证据链（5.3 执行层·首帧/新机位关键帧，见 `marked-camera-anchor-workflow.md`）**：全景权威/机位标记图(+`.annotations.json`)/干净机位底图/角色落位标记图/最终角色图齐全（缺证据 = `AUDIT_INCOMPLETE`）；能从近景遮挡/视差/距离压缩/左右关系**反推相机在标记原点**——只从旧全景转个向、原点没移到标记座位（如教室后方高处俯视全班）= 阻断；角色坐/站在标记支撑点、身体朝向符合箭头，冲突 = 阻断；**正式帧/宫格残留相机框/角色圈/箭头/CAMERA 标签未清除 = 阻断**；默认内置生图；**未经用户明确点名而启动外部/脚本生图（如 `parallel_image_gen.py`/小云雀）= 流程级 `FAIL_BLOCKED`：立即停止使用该输出、向用户披露、回到内置生图或 `WAITING_EXTERNAL_IMAGE` 路线，不得仅记录后继续宣称本闸 PASS**；
- **生图前置考量 8 维（§6.5）走全**：带主观视角/斜侧/过肩/多人站位/道具接触/跨帧连续的镜头逐帧声明了相关维度（含帧角色/动作时相、焦段≠机位移动、连续性入/出口）；简单平直镜头可标 `NA/沿用上一帧`。**仅当缺某维导致机位悬浮/站位乱/摆拍/道具漂移/光向跳变/动作时相不成立时 = 阻断（缺维本身不阻断，后果才阻断）**；
- **道具/产品**：形状/可读文字/持握手/接触点/承载面容器/相对位置/损坏或开盖状态无漂移；
- **连贯**：每帧从上一帧允许继承的状态接上，角色位置/地标侧/朝向/左右手/视线/光向/道具位置/承载面容器/地面痕迹/动势无跳变；
- **镜头与轴线**：画面方向、真实左右关系、视线、180° 轴线连贯，除非有可见的越轴/换位；
- **剧情忠实**：无新增剧情 beat/角色/地点/道具/台词、无改事件顺序；
- **宫格完整性**：顺序对、无 AI 重绘、关键动作没被裁掉、编号/标记不遮关键信息、Phase8 增补里有 artifact 排除守卫句。

**`FAIL_BLOCKED` 例**：因上一帧继承导致脸/服装设计漂移；**角色从书桌正面跳到侧面/与另一角色左右互换/坐站蹲状态变化，却无可见动作也没标"仅摄影机换位"**；**跨帧道具无可见搬移就换了承载面/容器（如铅笔盒早帧在桌上、后帧在纸箱里）**；宫格被 AI 重绘而非像素拼接；宫格引用进了上传顺序但 Phase8 Prompt 缺「仅供规划 / 不渲染 artifact」守卫句；当前角色/场景/状态变体/道具/产品缺权威图；需要 >6 张锚点却没拆 Clip；**机位悬浮于课桌过道/和任何座位都不对齐；为露侧脸/露手让角色转头歪脖摆拍，或把"侧脸必须可见"这类结果目标写进生图提示词正文**。

## 十一、最小回退
- 单帧漂移但素材正确 → 本阶段内重生/修该帧；
- **角色/道具无过渡空间漂移** → 先在本阶段补一帧可见动作（或在单帧写明"仅摄影机换位、角色不动"）重生；若该移动动作 Clip 表/Phase8 Prompt 根本就没有 → 回 Phase5 补动作 beat、再回来重生 8.5；
- **机位悬浮 / 摆拍 / 写了结果目标** → 本阶段重写机位规格块（把相机钉回物理锚 + 删结果目标 + 改"描述因"）重生；**若分镜本身要求了该机位几何上拍不到的可见域（如后排斜侧却要"完整侧脸"）→ 这是视频 Prompt 层的错，不走生图回灌；须在视频生成后经【闸10/人工审核】确认后、走 Phase8 自己的修正通道改分镜措辞（注明"视频层修改"），再回来重生 8.5**；
- **标记式两阶段失败分流（见 `marked-camera-anchor-workflow.md` 失败路由）**：相机原点/视差/眼高错 → 重做 Stage1；相机对、角色座位/朝向/人口错 → 重做 Stage2；帧基本对、仅一处手/道具/视线错 → Stage3 标记式局部返修（小红圈只修一处）；缺全景权威 → 回 Phase7；输出残留标记 → 带 artifact 守卫重跑同一 Stage；**相机+多角色+道具同时错时回 Stage1/2，不堆叠局部修**；
- 宫格拼接/版式/编号问题 → 本机重拼；
- Prompt 增补缺 artifact 守卫或上传槽位 → 修 Phase8/8.5 增补并重跑校验；
- 角色/场景/道具权威图缺失或错 → Phase7（素材从未登记则 Phase4）；
- Clip 需 >6 张锚点或动作连贯不清 → Phase5 Clip/连贯性回退，再重生 Phase8 Prompt 与 8.5 包。

## 与 storyboard-rag 的借鉴边界
**借鉴**：固定要素库思路、场景空间锁定、上一帧只继承连贯、动作/空间可执行性检查、越轴/反打站位一致性、5 格以内更适合即梦理解。
**不照搬**：不做黑白预演板、不做复杂导演说明板、不把宫格当人看的检查图、不塞参考区/箭头/文字说明、不沿用 storyboard-rag 执行层「禁止首帧/关键帧」的限制——本阶段目标就是生成这些视觉锚点（但守住"身份/画质归参考图、宫格只当构图锚"）。

## 触发（自动，非用户开口）

任一 Clip 成片到手后，**回报该 Clip「已接受」/ 进 Phase9 剪辑交接前必须先过本闸**。
- 含人物 / 复现道具 / 动作连贯 / 台词 / 视线 / 跨 Clip 关系的 Clip → 全维度审。
- 纯抽象 / 空场景 Clip → 仍查场景、时长、镜头、artifact 漂移。

## 三层架构

1. **Part A 确定性证据工具**（共享 `frame_audit.py`，**永不判分**）：解析 Prompt 集 → 全帧落盘 → 差分路由 → 分镜锚定取证帧 → 确定性硬核对 → 报告模板。
2. **Part B 视觉主判**：读取证帧，逐分镜逐维度填四态 + 证据。
3. **Part C 对抗性独立盲审**：派独立子代理盲审同一批取证帧（不给看主判结论），分歧逐格取更严一方。

## 前置完整性闸（先于四态质量判定）

工具状态：
- **`AUDIT_READY`**：全帧 + 差分 + 分镜映射都齐 → **证据齐、但未判定，≠PASS**。仅代表可以开始视觉判定。
- **`AUDIT_READY_WITH_DETERMINISTIC_RISK`**（`effective_status`）：证据齐，但命中确定性硬失败（截断/比例错）→ `deterministic_block=true`、`visual_verdict_floor=FAIL_BLOCKED`，**视觉判定地板就是 FAIL_BLOCKED**（见下节）。
- **`AUDIT_INCOMPLETE`**：任一项没齐（抽帧失败 / 只有样本帧 / 差分没跑 / 分镜没映射）→ **硬阻断**：禁 PASS、禁进剪辑、禁说 Clip 可用。先修工具 / 重抽再审，**不得弱化为 WARN**。

> ⚠ `AUDIT_INCOMPLETE` ≠ `FAIL_BLOCKED`：前者「证据没凑齐、还没资格下判」，返修=修工具/重抽；后者「视频内容有错」，返修=改视频/回退。

## 确定性硬核对（机检，可定 FAIL_BLOCKED）

工具产出 `deterministic_checks`：
- **时长** vs Prompt 标注：差 >0.3s 标 `RISK_OR_FAIL_REVIEW`；整段截断 / 动作缺失 → **FAIL_BLOCKED**。
- **比例 / 分辨率** vs Phase6 全局：超容差标 `RISK_OR_FAIL_REVIEW`；比例真错 → **FAIL_BLOCKED**（除非显式标「仅测试可接受」）。
- 首尾异常黑帧 / 纯色帧 → 视觉层确认。

> 🔴 **硬失败别被状态盖过（工具已机检显著化）**：命中硬失败时工具出 `effective_status=AUDIT_READY_WITH_DETERMINISTIC_RISK` + `deterministic_block=true` + `visual_verdict_floor=FAIL_BLOCKED`，报告头部显著显示——**视觉层必须按 `FAIL_BLOCKED` 处理**（除非显式接受测试风险），不能因头部 `AUDIT_READY` 就放行。比例容差按 `--audit-profile` 分档：`final`(默认)=1%、`test`=3%。

## 检查矩阵（逐分镜，每项写证据：帧@t / 实际观察 vs Prompt 原话 → 判定）

| 维度 | 判定要点 |
|---|---|
| 分镜对位 | 差分定位的真实切点与声明 `分镜N: a-b s` 偏差 >0.5s → FAIL |
| 人物朝向（重点） | **用参照物锚定法**（见下）判正对/背对/侧对，**不靠"被什么照亮"**；正/侧/背 + 画左/画右 对齐原话；写了转向要在入→出帧看到过程，不瞬切 |
| 站位漂移 | 左/中/右 + 前/后景 + **相对参照物（门窗桌等）**位置对齐；无动作位移 = 漂移 |
| 道具漂移 | 承载面/容器/归属手/开合/数量对齐；无搬动变位 = 漂移 |
| 跨镜一致 | 切点帧对：服装/伤痕/道具/站位/朝向/光向无解释跳变 → FAIL |
| 身份不跑脸 | 脸/发型/身材对齐 `@图片N`；多帧间不跑脸 |
| 动作节拍 / 口型 / 收尾 pose | 可见动作顺序匹配分镜无跳拍；有台词镜头嘴动同步；结尾姿态、多人转身次序符合 |
| 场景人口 / 光向 / 地标 | 该有背景群像的场景每帧都在、空镜确实无人；时段/光向/主要地标在位 |
| 锚点执行 | 成片帧 vs Phase8.5 该宫格构图/站位/朝向/道具 |

### 参照物锚定法（朝向 / 站位 / 跨镜 / 越轴 的统一判法，**防"被照亮=望向它"误判**）

不靠"人物在看哪 / 被什么照亮"主观判断，改成**锚定参照物 + 相对方位**四步：
1. **选参照物**：优先脚本为该镜头点名的朝向/视线锚点（窗/门/书桌/床/楼梯/货架/车/产品/另一具名角色）；没点名取画面中**固定、可辨、稳定**的地标。一镜至少锚一个；跨镜尽量锚**同一个**（用于查越轴/左右翻转）。能当参照物：固定空间地标 / 固定大道具 / 具名角色 / 产品——**会动的、临时的、光影不能当（光照 ≠ 方位）**；**相机术语（镜头/机位/运镜/景别）和手持可动小道具（铅笔/杯/手机）也不算参照物**（共享工具自动抽的"参照物锚点"槽目前可能误列这些，以固定地标为准，别拿"镜头"当朝向参照）。
2. **定参照物方位**：参照物在画面**哪个方位**——画左 / 画右 / 画面深处 / 人物正后方 / 画外某侧。
3. **判相对关系**：人物身体与视线相对该参照物是 **正对 / 背对 / 侧对（左·右）/ 俯 / 仰**。
4. **对脚本**：脚本"面向窗"=正对窗；实拍"背对窗"→ FAIL。🔴 **被参照物的光打亮 ≠ 正对它；参照物在身后 = 背对，哪怕抬头被它照亮也不算"望向它"。**

> 一并强化：站位（人物相对地标的左右/前后）、跨镜一致与越轴（相邻镜头人物相对**同一参照物**的左右关系若无机位解释翻转 = 越轴 FAIL）。
> **Clip2 反例（本规则缘由）**：参照物=窗（脚本点名）→ 窗方位=画面深处/人物正后方 → 人物身体正对镜头 = **背对窗**、视线向上向前未投向窗 → 脚本要正对窗 → **FAIL**（不能因被窗光打亮就算"望向窗"）。此误判我方主判与对抗盲审**双双漏过**，靠人工纠正——故视线目标在身后/轴外时，**人工抽查不能省**。

## 反糊弄判定协议

1. 默认存疑：每项先假设有漂移，拿正证据才判 PASS。
2. PASS 也要给正证据：禁「看起来一致 / 应该没问题」，必须帧 + 时间戳 + 实际观察 + 原话。
3. 怀疑项先穷举：每 Clip 先列所有「哪怕一点点不对」，再逐个裁。
4. 二元勾选表，不写散文。
5. 看不清只能降级（`ACCEPTED_WITH_RISK` / 待人工复看），**不准默认 PASS**。

## 对抗性独立盲审（Part C）

主判后派独立子代理，**只给取证帧 + 各分镜 Prompt 原话 + 对抗 brief**，不给看主判结论（防锚定）。brief 要点：默认有漂移、任务是找哪里对不上、死磕**人物朝向 / 切点帧对 / 道具归属手**、禁「基本一致 / 差不多」、落到可见像素。逐格比对，不一致回到该帧（必要时拉差分疑点附近全帧）重看，取更严一方或升级。报告保留两遍结论 + 分歧裁决。

## 五态与阻断

- 前置：`AUDIT_INCOMPLETE`（补证据，硬阻断）。
- 质量四态（前置过后才给）：`PASS / ACCEPTED_WITH_RISK(写明风险) / FAIL_BLOCKED / REGENERATED_PASS`。
- **阻断**：任一维度 `FAIL_BLOCKED` 或前置 `AUDIT_INCOMPLETE` 的 Clip → **不进剪辑**；未过本闸不得回报该 Clip「可用」。

## 判定 → 返修决策层（Repair Routing v1）

闸10 只判定；本层接住判定，把每个问题导向**最小处理路径**。两条**铁规则**贯穿全程：
- 🔴 **剪辑只能真修复、不能遮盖内容错**（插空镜/换切点/遮罩若只是把内容错藏起来→仍非 PASS）。
- 🔴 **内容重生成先做、容器批量后做**（重生成会作废先期的裁水印/对比例/换切点）。
- 总闸：命中即停，但**每降一档先证明"不伤剧情/连续/规格"**，证明不了就升级。

**固定顺序**
1. **证据** `AUDIT_INCOMPLETE` → 修工具/路径/抽帧/分镜映射；**不动 Prompt、不重生成、不剪**。审片不完整 ≠ 视频失败。
2. **内容先定稿**：一个 Clip 同时有内容错+容器错 → **先修内容**（否则裁了白裁）。
3. **容器/技术**（水印/黑边/比例/导出规格/头尾黑帧/**超容差**切点）→ Phase9 剪辑层，**交 `video-auto-edit` 批量执行**（裁/遮/重导/换切点）。**证明闸**：不损 脸/手/关键道具/口型/画面重心/动作落点，损了 → 升级重生成或按对比例重导。**比例首选按对比例重生成/重导**（根因是没按 Phase6 比例生成）；**容差内切点不返修**（不制造 make-work）。
4. **局部可删**：仅**非承重**才剪。**承重四问**（独占台词？独占动作因果？情绪落点？承接前后镜头的道具/站位/视线状态？）任一为是 → **不可剪**（对照 `{项目名}_故事板.md` 判，不靠感觉）。剪必写**时间码**（如 `00:03.708-00:04.125`），剪后复核 时长/比例/切点/**音画同步**/前后帧。
5. **内容烤进画面**（朝向/视线/站位/道具漂移/身份跑脸/场景人口/动作/跨镜断裂）→ 改 Prompt+重生成，**责任分流**：Prompt 含糊→改 Phase8（参照物锚定法当写法约束：窗位/身体朝向/头转向/视线目标写成几何关系）｜Prompt 清楚模型没做到→同 Prompt/加守卫句重抽，不改故事｜锚点/参考缺→回 Phase8.5/Phase7｜分镜缺 blocking/道具连续→回 Phase5｜人口/素材注册漏→回 Phase4/5。跨镜断裂→先判**上一 Clip 尾帧错还是下一 Clip 首帧错**（用切点帧对），必要时**成对处理两 Clip**。**朝向/视线默认不可剪**（除非该镜非承重）。重生成设**重试预算 K**：同 Prompt 重抽 ≤2 → Prompt 修版重生成 ≤2 → 仍失败**升级回 Phase5 重拆/拆 Clip**，不无限抽卡。
6. **接受风险**：仅用户明确接受 → `ACCEPTED_WITH_RISK`，写清风险，**不写 PASS**。

**闭环复检（按修法分档）**：重生成 → 该 Clip 全闸10重跑；重剪/换切点 → 切点对连续+时长+音画同步；跨镜成对 → 两 Clip+其边界；容器批量 → **全片 ffprobe + 抽样视觉复核**。

# 标记式两阶段机位锚点工作流（marked-camera-anchor-workflow）

> **定位**：Phase8.5 每个 Clip 的**首帧**、以及**引入新机位的关键帧**，用这套工作流——先把"场景权威图 → 验收过的干净机位底图"做出来，**再**加命名角色/背景人口/道具/动作。它是 §5.3「机位锚定图」从**文字规格**到**实际图像标记证据链**的执行升级：不只用文字描述相机在哪，而是真的在全景图上画标记、生成机位底图、再标记角色落位，留下可反推的证据。
> **性质**：本机工程工作流，**不是即梦官方原生规则**，别当平台规则讲给用户。
> **来源**：2026-06-21「重返那年盛夏」Clip4 用户确认的成功方法 → 通用化。验证案例只是验证，**不得把 Clip4 的场景/角色文件名硬编码成所有项目的默认**。

## 一、硬默认（两条红线 + 三条拆分原则）
1. **默认内置生图**：用户未明确点名脚本/小云雀/外部提供方时，出图走**本代理的内置图片生成/编辑能力**。Claude 侧若内置图片工具不可用，则**产出"标记图 + 无标记原图 + 准确提示词"，标 `WAITING_EXTERNAL_IMAGE` 等用户即梦/Codex 出图回贴**——**不要静默切到 `parallel_image_gen.py`、小云雀或第三方脚本**。**未经用户明确点名而启动外部/脚本生图 = 流程级 `FAIL_BLOCKED`：立即停止使用该输出、向用户披露、回到内置生图或 `WAITING_EXTERNAL_IMAGE`，不得仅记录后继续。**
2. **标记软引导优先于 Mask**：手动画的圈/框/箭头是软空间指令；**Mask/inpaint 不是首选**，只在用户明确要求、或标记式路线已失败并说明原因后才用。
3. **四阶段拆开，别让一次生成同时猜全部**：场景 / 机位 / 角色落位 / 局部返修分开做，不要一张图同时让模型猜教室结构+相机原点+角色座位+身体朝向+背景人口。
4. **标记脚本只画标记、不生图**：`scripts/mark_image_annotations.py` 纯离线（无网络、无生图代码，`generation_called: false`），只画框/箭头/圈/标签并存坐标。
5. **逐 Clip 暂停**：在 Phase8.5 正常确认闸停下，一次一个 Clip。

## 二、适用范围
- **Frame01**：永远走本工作流（Stage 0–2）。唯一例外：严格像素/连贯复用上一 Clip 已验收尾帧，且相机、站位、人口、道具、光线**全部不变**并在锚点计划留痕。
- **新机位**：任何新的相机物理坐标 / 眼高 / 视线方向 / 镜头位置关系 / 轴线侧 → **重做 Stage 1**。
- **同机位**：复用已验收的干净机位底图，**不重新生成空间几何**；只通过 Stage 2 加/改角色，小改动走 Stage 3 标记式局部返修。
- **缺干净全景**：回 Phase7 补一张。**不得**从角色身份板、裁切图或无关的早先关键帧里**反推**一个精确相机坐标。

## 三、Claude 生图路由（与 SKILL「本机不出图」对齐）
本机能做的两件确定性活：**① 用标记脚本画标记**（离线、可本地执行）；**② 本机像素拼宫格**（PIL/ffmpeg）。
出"机位底图/角色落位图/返修图"这一步是**生图**：
- 内置图片工具可用 → 直接调，输入"标记图 + 无标记原图（+ 角色/道具权威图）"。
- 内置图片工具不可用 → 产出可直接粘贴的提示词 + 已画好的标记图，标 `WAITING_EXTERNAL_IMAGE`，等用户即梦 / Codex 出图回贴，再继续 QC 与下一阶段。
- **任何情况下都不静默调用外部生图脚本 / 小云雀 / 优先 Mask**。

## 五、Stage 0：锁定空间全景
从 Phase7 场景资产选一张干净全景/宽景（暴露稳定地标 + 足够地面/家具几何来放相机），记录：前/后、真左/右；门/窗/固定家具/通道/座位网格坐标/180°轴；可用眼高选项与前景锚；时段/天气/光向/材质/人口基线。
> 全景是**空间权威**，不是最终机位画面，也不是人口指令。

## 六、Stage 1：标记机位 → 生成干净机位底图
1. 在**真实座位/站位/肩位/车辆座位**的眼位投影处画**相机框**（红框）。框画在**眼位**，不是桌面、不是被摄目标、不是箭头终点。
2. 从相机框画**一条箭头**指向视线目标。
3. 需要时加短标签 `CAMERA` 消歧。
4. 把"标记全景 + 无标记全景"交给内置生图（或产提示词），先要一张**不含命名角色的干净机位底图**。
5. 结果里**清除所有标记**。

核心提示词：
```text
第一张是带标记的空间图，第二张是无标记场景权威图。红框是相机所在的准确物理位置，不是拍摄目标；箭头是从该位置出发的镜头朝向。相机高度为{坐姿/站姿/肩位}眼平。生成从该位置真实看到的干净场景机位图。保持第二张的固定地标、空间结构、材质、时间和光线。最终不得保留框、箭头、文字或辅助线。
```
> 机位措辞坑（实证）：别只说"按标记机位看向箭头"，模型会误解观察原点。要锁死两个关系——**红框=相机所在位置、箭头=从该位置出发的朝向**；必要时补"不是这个视角，是从我标记的红框为机位、看向这里的视角"。**相机高度写"坐姿眼平/站姿眼平"，别用"眼高"二字**（"高"会被读成高角度俯视）。

**Stage 1 通过标准**：独立审查者能从**近景遮挡、地标视差、距离压缩、左右关系**反推出相机确实在标记原点。只是"朝目标大致转了个向、却仍保留旧全景原点（如教室后方高处俯视全班）" = `FAIL_BLOCKED`。

## 七、Stage 2：标记并加入角色 / 人口 / 道具 / 状态
1. 用验收过的 Stage 1 底图作**构图/几何锁**。
2. 在**真实支撑点**（椅子/地面标记/车座/床边/接触面）画各角色框（可用另一颜色，如绿色）。
3. 画**身体朝向箭头**；身体朝向不能决定视线时再加视线箭头。
4. 提供 Phase7 角色/状态权威图锁身份/服装/永久配饰。
5. 形状或标记要紧时提供 Phase7 道具/产品权威图。
6. 加入 Phase5/Phase8 明确的**人口状态**；背景人口须落在合理支撑点、保持场景朝向逻辑。
7. 交"标记底图 + 无标记底图 + 权威图"给内置生图（或产提示词）。

核心提示词：
```text
第一张是角色/动作位置标记图，第二张是已经验收的无标记机位底图，其余图片是角色和道具权威参考。保持第二张的相机位置、透视、地标、家具、光线和构图不变。把{角色}放在标记的{座位/站位}，身体沿箭头朝向{目标}，身份与服装只参考对应角色图；按已确认人口状态加入背景人物。标记只作空间指令，最终不得出现框、箭头、文字、UI或水印。
```
> Stage 1 底图只锁**几何**，**不能替代**角色/服装/道具权威图。

## 八、Stage 3：同机位关键帧与单点返修
- 同一相机坐标的动作中/末态：从验收过的 Stage 2 图出发，只标**变化的身体部位/道具接触点/视线目标/动作区域**。
- 每轮只修**一个连贯目标**。若相机几何、多角色位置、道具布局**同时错** → 回 Stage 1 或 2，别堆叠局部修。
- 交"标记图 + 干净已验收图"给内置生图；只为本次返修触及的身份/道具补权威图。
- **默认不用 Mask**；红圈/框/箭头是软指令、最终必须消失。

## 九、Stage 4：Gate9 与宫格
每帧验收前记录：场景全景权威；标记机位输入 + 干净 Stage 1 底图；标记落位输入 + Stage 2 结果；身份/道具权威；相机坐标/眼高/视线/前景锚/人口状态/允许变化；**所有标记已清除的正证据**。
每帧过 Gate9 后才拼宫格；**本机像素拼接**，绝不让生图模型重绘宫格。

## 十、失败路由
- 相机原点/视差/眼高错 → 重做 Stage 1。
- 相机对、角色座位/身体朝向/人口错 → 重做 Stage 2。
- 帧基本对、仅一处手/道具/视线错 → Stage 3 标记式局部返修。
- 身份/道具设计漂移 → 保几何，Stage 2/3 重新套 Phase7 权威图。
- 缺全景或身份/道具权威 → 回 Phase7。
- 输出残留标记 → 拒收，带 artifact 守卫重跑同一 Stage。

## 十二、与既有规则的关系
- 这是 `phase8-5-keyframe-anchors.md` §5.3「机位锚定图」的**执行层升级**：§5.3 写清相机锚的字段（相机身份/物理锚点/眼高/视线/前景锚/可见域涌现），本工作流把它落成"标记 → 机位底图 → 角色落位"的实际证据链。
- §5.1 道具位置图 / §5.2 角色站位图仍是跨帧空间锁；本工作流的 Stage 2 标记落实它们的"支撑点 + 身体朝向"。
- Gate9（`self-check-gates.md` 闸9）新增标记证据链阻断项；返修分流挂 VVK（`visual-verification-kernel.md`）。
